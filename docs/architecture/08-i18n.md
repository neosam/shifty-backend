# Internationalization (i18n)

Shifty supports three languages: **English (En), German (De),
Czech (Cs)**.

## Where Translations Live

In the frontend under `shifty-dioxus/src/i18n/` — presumably a
loader-based system with a key-value store per language.
**[To verify]** exact structure.

The backend itself does not return translated strings — it delivers
enum values or codes, and the frontend maps them to the user's
language. Exception: PDF templates and email templates (Text
Templates, F10) — those can be multilingual and are rendered in the
backend.

## Rule: Three Locales, No Exception

**[Convention]** (CLAUDE.md): Every new text key must be entered in
**all three languages** — not just En.

If you forget:

- Fallback either shows the key name or an empty string
  (**[To verify]** exact behavior).
- A Cs user sees German or English text — bad style if the other
  strings on the page are in Cs.

## Pattern for New Text

1. Introduce a new key in `i18n/en/…`.
2. Same key with German translation in `i18n/de/…`.
3. Same key with Czech translation in `i18n/cs/…` — when in doubt,
   machine-translate with a marker so a native speaker can review it.
4. Component uses a lookup like `t("my.key")`.

## Edge Cases

### Sentence Structure

German and Czech have different sentence order than English. If you
concatenate sentence fragments in the frontend
(`t("foo") + " " + t("bar")`), you break German and Czech.

**Rule:** Always use full-sentence templates with placeholders:

```
"welcome_message": "Willkommen, {name}, du hast {count} neue Nachrichten"
```

Never `"welcome" + user + "you_have" + count + "messages"`.

### Plural Forms

- English: 2 forms (1, ≠1).
- German: 2 forms (1, ≠1).
- Czech: 3 forms (1, 2-4, 5+ or 0).

**[To verify]** whether the i18n framework supports plural rules (ICU
MessageFormat or similar). If not, pluralizations have to be built
manually — when in doubt via an enum key with explicit forms.

### Number and Date Formats

Locale-dependent:

- German: `1.234,56` (decimal comma).
- English: `1,234.56` (decimal point).
- Czech: `1 234,56` (narrow space as thousands separator, decimal
  comma).

**[To verify]** how Shifty currently handles this — frontend
formatter or backend serialization.

### Feature-Specific i18n Keys

Some features (especially Reporting, Absence categories) have a large
number of enum values that all have to be translated. The feature
docs list the relevant key prefixes.
