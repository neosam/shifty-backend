# Phase 16 — UI Design Contract

> Visual and interaction contract for the Jahresansicht (year view) display phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** This is a *display extension* inside the existing Dioxus/Rust/WASM
> frontend (`shifty-dioxus`). There is **NO** shadcn, no Tailwind component-library
> introduction, and no new spacing/typography/color *system*. The existing
> `weekly_overview` table + chart, the token classes in `tailwind.config.js`, and the
> CSS custom properties in `input.css` ARE the design system and the source of truth.
> This contract specifies only the *deltas* this phase adds (a third token, a third
> chart segment, three new i18n strings) and pins the existing tokens those deltas must
> reuse.

## Color

This phase introduces **no new color tokens**. It assigns one EXISTING CSS custom property
to the new committed band. All three table tokens render as `text-ink` (the cell already
sets it); color differentiation in the *table* is carried by the emoji glyphs, not by text
color — consistent with today's `💰`/`🤝` tokens. Color differentiation in the *chart* is
carried by three distinct existing tokens.

| Role | Token | Usage |
|------|-------|-------|
| Dominant surface (60%) | `--bg` / `var(--surface)` (#fbfbfc / #ffffff) | Page + table background (existing) |
| Secondary (30%) | `var(--surface-alt)` (#f4f5f7) | Table header, current-row tint `bg-accent-soft` (existing) |
| Accent (10%) | `var(--accent)` (#3a4cd1) | **Paid** chart segment (full), current-week bar, nav focus (existing) |
| Committed band | `var(--good)` (#0e7a4d) | **Committed** chart segment — NEW assignment this phase |
| Surplus band | `var(--ink-muted)` (#6b7382) at opacity 0.35 | **Surplus/volunteer** chart segment (existing, unchanged) |
| Required marker | `var(--bad)` (#b8281a) | Dashed required-hours line in chart (existing, unchanged) |

**Chart stack order (bottom → top): paid (`--accent`) · committed (`--good`) · surplus
(`--ink-muted` @0.35).** Rationale: paid is the firmest commitment (full accent at the
base), committed is secured-but-voluntary (solid `--good` middle), surplus is the softest
band (muted gray top) — opacity/saturation decreases up the stack, preserving the existing
visual hierarchy where the top band is the dimmest.

**Committed segment color = `var(--good)`** (Claude's Discretion per D-04). Justification:
(1) it is an existing token → passes the `chart_uses_token_styles_not_legacy_hex` /
`page_source_does_not_use_legacy_classes` gates that forbid raw hex; (2) green semantically
reads as "secured/pledged capacity," visually distinct from accent-blue paid and muted-gray
surplus; (3) full opacity at the middle of the stack keeps the three bands clearly
separable. **HARD CONSTRAINT:** the committed segment MUST be expressed as
`background: var(--good);` (or another existing `var(--…)` token) — **never** a raw hex
literal. Forbidden hexes pinned by tests: `#3B82F6`, `#10B981`, `#EF4444`, `#e5e7eb`,
`#6b7280`, `#374151`.

Accent reserved for: paid chart segment (full), current-week bar fill, current-row table
tint (`bg-accent-soft`), and nav-button focus ring. **Not** used for the committed band.

Dark-theme note: all chosen tokens (`--accent`, `--good`, `--ink-muted`, `--bad`) have
dark-theme values in `input.css:54-77`; using tokens (not hex) means dark mode works
automatically. No extra work.

## Token Display Semantics (locked by D-02 / D-03)

The table cell shows **three separate tokens, never combined**:

```
💰{paid_hours}  |  🎯{committed_voluntary_hours}  |  🤝{volunteer_hours}
```

- `🎯` = committed (Band 1, the pledge). Emoji choice is Claude's Discretion (`🎯`
  confirmed — a target/goal glyph reads as "what was pledged").
- `🤝` = surplus over the pledge (Band 2), the **existing** volunteer token, semantics
  narrowed by Phase 15 but glyph unchanged.
- Worked example `committed=5, actual=7` → renders `💰{paid} | 🎯5.00 | 🤝2.00`.
- Worked example `committed=5, actual=3` (under pledge) → renders `💰{paid} | 🎯5.00 | 🤝0.00`.
- **Zero rule (D-03):** `committed == 0` renders `🎯0.00` — plain zero, two decimals,
  identical to how paid/surplus already render zero. **No** blank/dash/em-dash
  special-casing. The blank-vs-zero idea is explicitly out of scope (Phase 17, employee
  view only).

All three values use `format_hours(value, 2)`.

## Copywriting Contract

This is a read-only display phase: there is no CTA, no form, no empty/error/destructive
state introduced. The "copy" surface is the i18n label set (CVC-08). All three locales
(De/En/Cs) MUST be populated simultaneously — the `Locale::De`-vs-`Locale::En` swap bug
(`shifty-dioxus/CLAUDE.md`) is the named hazard; guard with per-locale reference-matcher
tests.

| Element | De | En | Cs |
|---------|----|----|-----|
| `Key::Committed` (token + chart legend label) | `Zugesagt` | `Committed` | `Přislíbeno` |
| Table header — three-band (NEW `Key::PaidCommittedVolunteer`) | `Bezahlt / Zugesagt / Freiwillig` | `Paid / Committed / Volunteer` | `Placené / Přislíbeno / Dobrovolné` |

**Czech translations are MEDIUM-confidence** (derived from project vocabulary;
`Přislíbeno` = "pledged/promised", `Dobrovolné` aligns with existing
`CategoryVolunteerWork = "Dobrovolnictví"` / `ShowPaid = "Placené"`). User should confirm
during review (RESEARCH Assumption A3).

**Header key decision (Claude's Discretion, resolved):** add a NEW key
`Key::PaidCommittedVolunteer` rather than mutating `Key::PaidVolunteer`. The existing
`Key::PaidVolunteer` ("Paid / Volunteer") stays defined so its name remains
self-describing and any other consumer is unaffected. The table header at
`weekly_overview.rs:58` switches from `Key::PaidVolunteer` to
`Key::PaidCommittedVolunteer`.

**Pre-existing i18n gaps this phase MUST also close (verified at HEAD):**

| Key | de.rs | en.rs | cs.rs | Action |
|-----|-------|-------|-------|--------|
| `Key::Volunteer` | present (572) | present (517) | **MISSING** | add `Dobrovolné` to cs.rs |
| `Key::PaidVolunteer` | present (133) | present (105) | **MISSING** | add `Placené / Dobrovolné` to cs.rs |

| Standard element | Status |

| Primary CTA | none (read-only display phase) |

| Error state | not applicable — no new fetch/error path introduced |

## Interaction Contract

No new interactions. The third token and segment are **display-only** and inherit existing
behavior:

| Element | Behavior (existing, unchanged) |
|---------|--------------------------------|
| Chart bar click | navigates to `/shiftplan/{year}/{week}` |
| Chart bar hover | native `title` tooltip — now lists three values incl. committed |
| Current-week bar | full accent fill; other bars dimmed (`opacity: 0.85`) |
| Current-week row | `bg-accent-soft` tint |
| Required line | dashed `var(--bad)` marker — unchanged position/style |

The diff column (`available_hours − required_hours`) is unchanged in the frontend;
`available_hours` now carries the committed term because the BACKEND computes
`overall_available_hours = paid + committed + surplus` (D-01). The chart bar total MUST be
recomputed to match (`bar_total = paid + committed_voluntary_hours + volunteer`,
`weekly_overview_chart.rs:16`) or the bar height will silently diverge from the diff column
(RESEARCH Pitfall 3).
