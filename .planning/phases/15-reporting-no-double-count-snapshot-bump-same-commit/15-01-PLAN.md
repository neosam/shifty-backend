No-double-count invariant: per person/week `committed + max(actual − committed, 0) = max(committed, actual)` — the two bands never overlap.

## Multi-person aggregation: ADOPT FORMULA B — two-band decomposition (D-05, supersedes Formula A)

Per 15-CONTEXT.md **D-05** (user clarification 2026-06-23) and 15-VALIDATION.md "✅ Semantic decision RESOLVED — D-05", this plan adopts **FORMULA B = per-person `max`**, decomposed into two stacked bands. This OVERRIDES the earlier "single max-of-sums term / Formula A / multi_person = 5.0" framing that the prior draft pinned.

```
Band 1  committed_voluntary_hours = Σ_week Σ_person committed_p            (cap-gated, flat — D-03)
Band 2  volunteer_hours           = Σ_week Σ_person max(actual_p − committed_p, 0)   (cap-gated subtraction)
```

Per person/week the two bands sum to `committed_p + max(actual_p − committed_p, 0) = max(committed_p, actual_p)` — the no-double-count closed form, with the `max` taken PER PERSON (never on aggregates).

Rationale (document this in the SUMMARY):
1. **Person-set overlap is real (user-confirmed):** an `is_paid=false` volunteer CAN have `committed_voluntary` set. Therefore Band 2's surplus MUST be computed per person — you cannot subtract aggregate-from-aggregate, because `max(Σactual − Σcommitted, 0) ≠ Σ max(actual_p − committed_p, 0)` when persons differ.
2. **Band 1** reuses `committed_voluntary_for_calendar_week` (which already sums committed across active persons' rows for the week), wrapped in a cap filter (CVC-06).
3. **Band 2** iterates the per-person volunteer reports already produced at booking_information.rs:141-153 (each gives `(sales_person_id, actual_p)`), subtracting that person's own per-week committed (cap-gated), floored at 0.
4. For `is_paid=true` capped persons `actual_volunteer = 0` in Achse B (RESEARCH "Critical Open Question" → Option b: their actuals live in `paid_hours`/Achse A) ⇒ Band 2 = 0, Band 1 = committed.

The `multi_person` fixture (Plan 15-02) is therefore PINNED to **committed_voluntary_hours = 5.0, volunteer_hours = 3.0, total = 8.0**: Person A (cap=true, committed=5, actual=0) → Band1 += 5, Band2 += max(0−5,0)=0; Person B (cap=false→committed gated to 0, actual=3) → Band1 += 0, Band2 += max(3−0,0)=3.

This is the single highest-risk decision of the phase. It is encoded here so Plan 15-02's fixtures inherit FORMULA B.
