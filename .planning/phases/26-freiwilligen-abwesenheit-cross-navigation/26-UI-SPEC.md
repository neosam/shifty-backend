## Color

All values are CSS-variable aliases; no raw hex values are used. Tokens are
pre-existing and enforced by the 18 self-policing `no_legacy_classes_in_source`
tests. NAV-01 adds no new color usage.

| Role | Tailwind token | Usage |
|------|---------------|-------|
| Dominant surface (60%) | `bg-surface` | Page background — unchanged |
| Secondary (30%) | `bg-surface-alt` | Cards, filter bar — unchanged |
| Accent (10%) | `text-accent` / `bg-accent-soft` | Reserved for: primary CTA button labels, active filter indicators, inline text links |
| Destructive | `text-bad` / `bg-bad-soft` | Delete actions only — NAV-01 has none |

**NAV-01 color usage:** Ghost buttons inherit `text-ink` at rest, `hover:bg-surface-alt`
on hover — the existing `BtnVariant::Ghost` appearance. No new color usage.

**Forbidden classes** (project lint rule): `bg-gray-*`, `bg-white`, `text-gray-*`,
`text-blue-*`, `text-red-*`, `text-green-*`, `bg-blue-*`, `bg-green-*`, `bg-red-*`,
`border-black`, `border-gray-*`.

Accent reserved for: primary CTA button (new absence), active filter/sort indicators,
inline text action links (edit, convert). NAV-01 ghost buttons do NOT use accent — they
are secondary navigation, not primary actions.

## Interaction Contract

### New Route Variant

Add to `src/router.rs` `Route` enum:

```
#[route("/absences/:employee_id/")]
AbsencesFor { employee_id: String },
```

Add alias in `src/router.rs` module-level use block:

```
pub use crate::page::AbsencesFor;
```

The route `/absences/` (param-less, existing) is unchanged.

The new variant renders a thin wrapper component `AbsencesFor` in
`src/page/absences.rs` (add below `AbsencesPage`). The wrapper:
1. Parses `employee_id: String` as `Uuid`. If parsing fails, renders an inline
   error message (`"Invalid employee id"`, no separate error page).
2. On mount, seeds the `person_filter` signal via a `GlobalSignal` or passes the
   pre-selected UUID as a signal seed into the page render. Simplest approach:
   add a module-level `pub static ABSENCES_PRESELECT: GlobalSignal<Option<Uuid>>`
   to `src/page/absences.rs` (analogous to `EMPLOYEES_LIST_REFRESH` pattern).
   `AbsencesFor` writes to it on mount; `AbsencesPage` reads it once on first
   render to initialise `person_filter`.
3. The pre-seeded person_filter is mutable — the user can still change the
   selection via the existing filter dropdown.
4. The route is bookmarkable (D-26-05 decision).

### Link 1 — MyEmployeeDetails → Absences (Sales role)

**Source file:** `src/page/my_employee_details.rs`

**Placement:** first child inside the outer `div { class: "ml-1 mr-1 pt-4 md:m-8" }`,
before the existing `EmployeeView` component call.

**Render:**
```rust
let nav = use_navigator();
let i18n = I18N.read().clone();
// ...existing component code...
rsx! {
    TopBar {}
    ErrorView {}
    // ...modals...
    div { class: "ml-1 mr-1 pt-4 md:m-8",
        // NAV-01 START
        div { class: "mb-4 flex items-center",
            Btn {
                variant: BtnVariant::Ghost,
                on_click: move |_| { nav.push(Route::Absences {}); },
                "{i18n.t(Key::NavToMyAbsences)}"
            }
        }
        // NAV-01 END
        EmployeeView { ... }
    }
}
```

**When shown:** always (Sales role always has access to own absences).

**Style:** `BtnVariant::Ghost` — renders with `text-ink` text, no background,
`hover:bg-surface-alt`. No icon. No arrow suffix in label.

**Navigator:** imperative `use_navigator().push(Route::Absences {})`.

### Link 2 — EmployeeDetails → AbsencesFor (HR role)

**Source file:** `src/page/employee_details.rs`

**Placement:** inside `div { class: "px-4 py-4 md:px-8 md:py-6 flex flex-col gap-4" }`,
after the `if is_mobile { Btn Ghost back }` block, before `EmployeeView`. The
existing `flex flex-col gap-4` spacing already separates elements correctly.

**Render:**
```rust
Btn {
    variant: BtnVariant::Ghost,
    on_click: move |_| {
        nav.push(Route::AbsencesFor {
            employee_id: employee_id.to_string(),
        });
    },
    "{i18n.t(Key::NavToEmployeeAbsences)}"
}
```

**When shown:** always visible on EmployeeDetails page (HR always sees the link
to that employee's absences). No additional condition beyond "on this page".

**Navigator:** same `use_navigator().push(...)` pattern. `employee_id` is already
a `Uuid` in scope at render time.

### Link 3 — AbsencesPage (Sales) → MyEmployeeDetails

**Source file:** `src/page/absences.rs`, `AbsencesPage` component.

**Placement:** inside `div { class: "p-4 md:p-6 flex flex-col gap-3" }`, as the
**first child**, before the `header` element. The existing header renders the page
title and primary CTA and must not be modified.

**Render:**
```rust
if !is_hr {
    div { class: "flex items-center",
        Btn {
            variant: BtnVariant::Ghost,
            on_click: move |_| { nav.push(Route::MyEmployeeDetails {}); },
            "{i18n.t(Key::NavToMyTimeAccount)}"
        }
    }
}
```

**When shown:** only when `!is_hr`. On the employee (Sales) self-view, the link
is always present — no dependency on filter state.

### Link 4 — AbsencesPage (HR, person selected) → EmployeeDetails

**Source file:** `src/page/absences.rs`, `AbsencesPage` component, same placement
as Link 3.

**Render:** rendered in the same `if !is_hr { ... }` / `else { ... }` block:
```rust
if is_hr {
    if let Some(selected_id) = person_filter_val {
        div { class: "flex items-center",
            Btn {
                variant: BtnVariant::Ghost,
                on_click: move |_| {
                    nav.push(Route::EmployeeDetails {
                        employee_id: selected_id.to_string(),
                    });
                },
                "{i18n.t(Key::NavToEmployeeReport)}"
            }
        }
    }
}
```

**When shown:** only when `is_hr && person_filter_val.is_some()`. When no person
is selected in the filter, the link is absent (no empty placeholder, no disabled
state). This is consistent with "link is only meaningful when context is clear".

**State dependency:** reads `person_filter_val` (the current `*person_filter.read()`
snapshot at render time). The link updates reactively as HR changes the person
selector.
