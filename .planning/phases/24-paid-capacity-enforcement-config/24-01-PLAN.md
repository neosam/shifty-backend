<objective>
Establish the backend error contract and storage seed that the rest of Phase 24 builds on:
1. A new, distinct `ServiceError::PaidLimitExceeded { current: u8, max: u8 }` variant (D-24-05).
2. Its central HTTP mapping to a **non-403** status — **409 Conflict** — so the frontend can surface it (403 is silently dropped by the frontend, D-13).

Purpose: This plan is the producer for the contract that the backend enforcement (24-02) and the frontend inline-block handling (24-05) both depend on. It contains no business logic — it only adds the error variant, its status code, and the toggle seed.
Output: New ServiceError variant, 409 mapping, OpenAPI response annotation update, new migration file.
</objective>
