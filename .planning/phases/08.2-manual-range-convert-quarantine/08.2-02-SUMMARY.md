# Phase 8.2 Plan 02: Manual-Convert Modal (Frontend) Summary

**Frontend ManualConvertModal replaces the 8.1-09 EditExtraHoursModal stub and consumes the Plan 08.2-01 backend `manual_range` branch via a new `CutoverAction::ConvertSingleManualRange` coroutine variant. The modal renders 2× `<input type="date">` plus read-only amount/category context (D-31/D-32), pre-fills both date strings with the quarantine row's date, validates start ≤ end inline (P-7 defense — no hardcoded fallback date), and dispatches the action with parsed `time::Date` boundaries. The coroutine forwards the request via `api::cutover_convert_quarantine_entry(..., Some(ManualRangeTO))`, applies the P-6 fallback (separate `gate_dry_run` if backend's inline replay returned `None`), writes the refreshed drift report to `CUTOVER_STORE.last_dry_run`, and bumps `CUTOVER_DRIFT_REFRESH`. 8 new i18n keys ship across DE/EN/CS with the per-locale reference-matcher test extended to fang Pitfall-2 (Locale::En instead of Locale::De). 4 new dioxus-ssr snapshot tests replace the obsolete EditExtraHoursModal Test 11. WASM build (`cargo build --target wasm32-unknown-unknown`) is green; the full shifty-dioxus binary test suite (536 tests) passes 0 failures. Karin-Pattern (gap-1a) is now operatively resolvable end-to-end.**

### Modal Component (`shifty-dioxus/src/page/cutover_admin.rs`)

`EditExtraHoursModal` (Z. 817-897 in the 8.1-09 file) is replaced fully by `ManualConvertModal`. Component contract:

```rust
#[component]
fn ManualConvertModal(
    entry: EntryRef,
    category: AbsenceCategoryTO,
    on_submit: EventHandler<(Uuid, time::Date, time::Date)>,
    on_cancel: EventHandler<()>,
) -> Element
```

Rendered surface:

- **Custom-Backdrop** (existing 8.1-09 pattern reused 1:1): `fixed inset-0 bg-modal-veil flex items-center justify-center z-50` + outer `onclick = on_cancel.call(())` + inner `onclick = ev.stop_propagation()`.
- **Title + help paragraph** via `CutoverManualConvertModalTitle` + `CutoverManualConvertHelp`.
- **Read-only context row** (D-31 + D-32): amount as `<span class="font-mono">{amount:.2}h</span>`, category label resolved via STATIC Tailwind match arms (`AbsenceCategoryTO::Vacation/SickLeave/UnpaidLeave` → `Key::AbsenceCategoryVacation/SickLeave/UnpaidLeave`), weekday in muted text. NO `<input>`/`<select>` for amount or category.
- **Two date inputs**: `<input r#type="date" value="{start_str}" oninput=... />` for start + end. Both signals pre-filled with the quarantine row's date.
- **Inline error**: `if let Some(e) = error_msg.read().clone() { span { class: "text-bad text-small", "{e}" } }`.
- **Footer**: Cancel button (reuses `Key::CutoverEditBtnCancel` per RESEARCH § 6) + Submit button with the new `Key::CutoverManualConvertBtnSubmit` label.
- **Submit handler** (P-7 Defense — verbatim from RESEARCH § Code-Examples):
  ```rust
  match (Date::parse(start_str, fmt), Date::parse(end_str, fmt)) {
      (Ok(s), Ok(e)) if s <= e => { error_msg.set(None); on_submit.call((entry_id, s, e)); }
      (Ok(_), Ok(_))           => { error_msg.set(Some(err_start_after_end.clone())); }
      _                        => { error_msg.set(Some("Invalid date format".to_string())); }
  }
  ```
  No `unwrap_or_else(|_| date!(2026-01-01))` — exactly the 8.1-09 smell the plan body called out.
