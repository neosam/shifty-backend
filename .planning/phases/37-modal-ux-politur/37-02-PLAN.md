<objective>
Fix MOD-02: the Arbeitsvertrag (employee work details / contract) modal shows a short, per-field explanatory sentence under each relevant field, following the existing `CapPlannedHoursHelp` pattern already present at `contract_modal.rs:427`. Von/Bis (From/To) are excluded (self-explanatory, D-06).

Approach (D-05): render each help text as a sibling `<span class="text-small font-normal text-ink-muted">` next to its field — the minimal, consistent mirror of the current cap help. Do NOT restructure the shared `Field` atom into a help slot (the Field atom's built-in `hint` prop is `text-micro` and is intentionally NOT used here).

Each new help text is a new i18n key present in all three locales (D-09): a `Key` enum variant in `i18n/mod.rs` placed next to its `*Label`, plus an `add_text(...)` in `de.rs`, `en.rs`, and `cs.rs`. All three are mandatory — a missing locale silently falls back to German (CLAUDE.md). German texts are taken verbatim from the D-06/D-07 table (no rewording); English and Czech are translated (Claude's discretion, faithful to the German meaning).

Purpose: make the contract form self-documenting so HR users understand each field without external docs.

Output / Artifacts this phase produces:
- Six new `Key` variants: WorkdaysHelp, ExpectedHoursPerWeekHelp, DaysPerWeekHelp, VacationEntitlementsPerYearHelp, DynamicHourHelp, CommittedVoluntaryHelp.
- 18 new `add_text` translations (6 keys × de/en/cs).
- Six sibling help spans wired into contract_modal.rs, each under its field.

No new attack surface: static, localized help copy rendered client-side; no user input, no network, no persisted data. No new dependencies, no migration, no snapshot bump.
</objective>
