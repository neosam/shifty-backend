<objective>
Erweitere `AbsenceService` um die Methode `derive_hours_for_range(from, to, sales_person_id, ctx, tx) -> Result<BTreeMap<time::Date, ResolvedAbsence>, ServiceError>`. Der Resolver wendet Cross-Category-Prioritaet `SickLeave > Vacation > UnpaidLeave` (D-Phase2-01..03, BUrlG §9-konform) an, nutzt per-Tag den am jeweiligen Tag gueltigen `EmployeeWorkDetails`-Vertrag fuer Stunden-Berechnung, und setzt an Feiertagen 0 Stunden. AbsenceService erhaelt `SpecialDayService` und `EmployeeWorkDetailsService` als neue Deps.

Purpose: Phase 2 Single source of truth fuer Phase-4-Migration-Gate. Der Resolver lebt im AbsenceService (D-Phase2-02), nicht im ReportingService — das vermeidet Duplikation der BUrlG-§9-Logik in Reporting-Switch und Migration-Validation.

Output: 1 Trait-Methode (PUB API) + 1 Domain-Struct + 1 Impl-Funktion + 2 neue Deps + 3 gruene Tests + DI-Wiring.

   ```rust
   async fn derive_hours_for_range(
       &self,
       from: time::Date,
       to: time::Date,
       sales_person_id: Uuid,
       context: Authentication<Self::Context>,
       tx: Option<Self::Transaction>,
   ) -> Result<BTreeMap<time::Date, ResolvedAbsence>, ServiceError> {
       use shifty_utils::DateRange;

       // Permission: HR oder self (analog find_by_sales_person)
       // Pruefe in der bestehenden find_by_sales_person-Implementation wie das gemacht wird,
       // und uebernimm das Pattern 1:1.
       // ... permission check copy from find_by_sales_person ...

       let tx = self.transaction_dao.use_transaction(tx).await?;

       // 1. Batch-Fetch Absences
       let absences = self.absence_dao
           .find_by_sales_person(sales_person_id, tx.clone()).await?;

       // 2. Batch-Fetch WorkDetails
       let work_details = self.employee_work_details_service
           .find_by_sales_person_id(sales_person_id, context.clone(), Some(tx.clone())).await?;

       // 3. Kalenderwochen-Set aus Range
       let date_range = DateRange::new(from, to)?;
       let mut weeks: std::collections::BTreeSet<(u32, u8)> = std::collections::BTreeSet::new();
       for day in date_range.iter_days() {
           let iso = day.iso_week();  // gibt (year, week) — pruefe time-API
           weeks.insert((day.year() as u32, day.iso_week() as u8));
       }

       // 4. Holidays sammeln
       let mut holidays: std::collections::BTreeSet<time::Date> = std::collections::BTreeSet::new();
       for (year, week) in weeks.iter() {
           let special = self.special_day_service
               .get_by_week(*year, *week, context.clone()).await?;
           for sd in special.iter() {
               if matches!(sd.day_type, service::special_days::SpecialDayType::Holiday) {
                   holidays.insert(sd.date);
               }
           }
       }

       // 5. Per-Tag-Resolver
       let mut result = BTreeMap::new();
       fn priority(c: &AbsenceCategory) -> u8 {
           match c {
               AbsenceCategory::SickLeave => 3,
               AbsenceCategory::Vacation => 2,
               AbsenceCategory::UnpaidLeave => 1,
           }
       }
       for day in date_range.iter_days() {
           // Active contract for this day
           let active_contract = work_details.iter().find(|wh| {
               // Pruefe wh.from_date()/to_date()-Pattern; falls Result<>: .ok() handlen
               // Pseudo: wh.from_date() <= day && day <= wh.to_date()
               match (wh.from_date(), wh.to_date()) {
                   (Ok(start), Ok(end)) => start <= day && day <= end,
                   _ => false,
               }
           });
           let Some(contract) = active_contract else { continue };
           if !contract.has_day_of_week(day.weekday()) { continue }
           if holidays.contains(&day) { continue }
           let hours = contract.hours_per_day();
           if hours <= 0.0 { continue }

           // Active absences fuer day
           let mut active: Vec<&service::absence::AbsencePeriod> = absences.iter()
               .filter(|ap| ap.deleted.is_none() && ap.from_date <= day && day <= ap.to_date)
               .collect();
           if active.is_empty() { continue }
           active.sort_by_key(|ap| std::cmp::Reverse(priority(&ap.category)));
           let dominant = active[0];
           result.insert(day, ResolvedAbsence {
               category: dominant.category.clone(),
               hours,
           });
       }

       self.transaction_dao.commit(tx).await?;
       Ok(result)
   }
   ```
   **Hinweise:**

   - `time::Date::iso_week()` gibt einen `u8` zurueck; `time::Date::year()` gibt `i32` (cast nach u32)
   - `EmployeeWorkDetails::from_date()`/`to_date()` gibt vermutlich `Result<ShiftyDate, _>` — pruefe `service/src/employee_work_details.rs` und passe das `.ok()`/`?`-Handling an
