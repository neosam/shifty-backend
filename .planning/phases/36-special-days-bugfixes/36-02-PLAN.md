<objective>
Fix SDF-02: in the Settings Special-Days card (Card-3), after creating the first holiday the Anlegen button stays disabled because the `<select>` is uncontrolled — `sd_type.set(None)` resets the signal but the DOM still shows the previously chosen option, so `sd_form_valid` reads `None` and the button never re-enables without toggling the dropdown.

Purpose: Fix the root cause (D-05) by making the shared `SelectInput` support an optional controlled `value` prop, then bind Card-3's `<select>` to the `sd_type` signal (D-06). This keeps signal and DOM in sync for every dropdown, immune to future desync, while staying backward compatible for all existing callers (D-07).

Output: A controlled-capable `SelectInput`, a wired Settings Card-3 select, a small pure mapping helper, and cargo/SSR regression tests. No new user-visible text.
</objective>
