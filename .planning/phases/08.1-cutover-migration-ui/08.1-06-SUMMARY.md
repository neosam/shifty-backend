## Czech Translation Rationale

Czech strings were chosen by direct German→Czech translation where the term is unambiguous in HR/admin contexts (e.g., `Datenmigration` → `Migrace dat`, `Übersicht` → `Přehled`, `Weiter` → `Pokračovat`, `Zurück` → `Zpět`). Style aligns with the existing `cs.rs` absence-domain block (Plan 08-04): no diacritic shortcuts, full forms (e.g., `nepřítomnosti`/`migrace`), HR-tone consistent with neighbouring keys. Reference samples pinned by `i18n_cutover_keys_match_czech_reference`:

| Key | Czech |
|-----|-------|
| `CutoverPageTitle` | Migrace dat |
| `CutoverStage1Label` | Přehled |
| `CutoverBtnContinue` | Pokračovat |
| `CutoverBtnBack` | Zpět |
| `CutoverCommitBtn` | Provést cutover |

## Decisions Made

1. **Single-line `add_text` style for the new Cutover block** — the verify-grep `grep -c 'add_text(Locale::X, Key::Cutover'` only matches one-line calls. Multi-line (rustfmt-default) splits the call across lines and the count drops below 34. Inline comment placed above the block warns future rustfmt runs not to wrap it. The existing absence-domain block has mixed multi-line layout, but its plan-verify count was historically lenient; for 8.1-06 the Plan-08.1-06 verify is strict (`returns 34`), so single-line is enforced for Cutover only.
2. **Czech sourced from German references where unambiguous** (Plan-frontmatter guidance) — keeps tone consistent with the existing `cs.rs` absence-domain block. Pinned by the 5-key `i18n_cutover_keys_match_czech_reference` test.
3. **`CutoverMenuLabel` (TopBar Verwaltung-Submenu) included in this plan** rather than deferred to Plan 07 — central i18n declaration block keeps key registration atomic; Plan 07 consumes the key without needing to touch i18n files.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Multi-line `add_text` calls broke plan-verify grep**
- **Found during:** Task 2 (de.rs initial write)
- **Issue:** The first draft used rustfmt-style multi-line `add_text(\n  Locale::De,\n  Key::CutoverPageSubtitle,\n  "...",\n);` for long-string calls. The plan's `<verify>` and `<done>` criteria require `grep -c 'add_text(Locale::De, Key::Cutover'` to return exactly 34, but multi-line calls only match the first line (no trailing key on the same line) — actual count was 17 instead of 34.
- **Fix:** Rewrote all Cutover `add_text` calls as single-line (one statement per line, full args inline). Added inline comment above the block warning against rustfmt reformatting.
- **Files modified:** shifty-dioxus/src/i18n/de.rs (then mirrored in en.rs and cs.rs from the start).
- **Verification:** `grep -c` returns 34 for each of de.rs/en.rs/cs.rs.
- **Committed in:** `75daa247` (Task 2 commit), `04fd0d7c` (Task 3 commit), `b7c0220d` (Task 4 commit).

**2. [Rule 1 - Bug] Inline comment containing the grep pattern matched itself**
- **Found during:** Task 2 (after the single-line rewrite)
- **Issue:** First-draft comment "// Plan 08.1-06 verify grep `grep -c 'add_text(Locale::De, Key::Cutover'`" itself matched the verify-grep, pushing the count to 35.
- **Fix:** Reworded the comment to drop the grep pattern from its body — it now reads "// Cutover migration (Phase 8.1) — single-line calls; do not reformat across multiple lines (Plan 08.1-06 verify grep counts call lines)."
- **Files modified:** shifty-dioxus/src/i18n/de.rs (en.rs and cs.rs were authored after the fix and are clean).
- **Verification:** `grep -c 'add_text(Locale::De, Key::Cutover' de.rs` returns 34.
- **Committed in:** `75daa247` (Task 2 commit, pre-commit fix).

**3. [Rule 3 - Blocking] WASM linker `lld` not on system PATH**
- **Found during:** Final verification (post-Task 4)
- **Issue:** `cargo build --target wasm32-unknown-unknown` failed with "linker `lld` not found". This is a NixOS environment specifics — `lld` is gated behind `nix-shell -p lld`.
- **Fix:** Re-ran the WASM build via `nix-shell -p openssl pkg-config lld --command "cargo build --target wasm32-unknown-unknown"`. No source change needed; environmental fix only.
- **Files modified:** _none_ (environment-only).
- **Verification:** WASM build exits 0, all warnings only.
- **Committed in:** _N/A_ (no source change; documented here as deviation per protocol).
