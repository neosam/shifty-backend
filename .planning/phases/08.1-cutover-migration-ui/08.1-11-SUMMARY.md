## Decisions Made

1. **Re-use 08-HUMAN-UAT.md structural template** — same frontmatter, same per-step `expected/result` micro-block, same Summary + Gaps + Sign-off blocks. Rationale: cognitive consistency for the same tester across two UATs (this one + Plan 12 Phase-8-Subsumption). The plan-body's exact step content (the 9 sections with the exact action-button labels, seed-data instruction, and tooltip-key references) is preserved 1:1; only the rendering shell changed.

2. **Section-0-Setup separated from `total` count** — The 5 Setup checkboxes (0.1-0.5) are pre-conditions: env-readiness, user accounts, feature-flag pre-state, optional seed, dev-tools open. These don't pass or fail in the same sense as the actual UAT steps. Counting them in `total` would dilute the percent-passed metric.

3. **Stage-3-gate verification accepts both UI-presentations** (Step 1.4) — Either "Weiter is disabled at Stage 2" OR "Stage 3 reachable but Commit-Btn disabled with tooltip" is acceptable. Plan 08.1-09's actual implementation gates at the commit button (`disabled = !matched || !is_admin || quarantined > 0`); the Stage 3 tile is reachable. The contract is the inability to commit, not which UI element shows the disabled state.

4. **Seed-data instruction is concrete** (Step 0.4) — Tester is told to insert a fractional Vacation entry (e.g. 13.33h) if `quarantined_rows == 0`. Plan 08-09's `detect_weekly_lump_sum` rejects fractional amounts (returns `None`); Plan 08-08's `compute_gate` flags such rows as `FractionalAmount`. This makes the seed deterministic and aligned to backend behaviour without requiring SQL-shell access.

5. **Type-to-confirm verified via three sub-steps** (6.3 / 6.4 / 6.5) — Lowercase, typo-suffix, and exact-match are tested independently. Single "enter the string" check would miss case-sensitivity bugs.
