## Spacing Scale

All values are multiples of 4px. Sourced from existing Tailwind token usage in `absences.rs` and `btn.rs` (Phase 8 baseline).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline badge padding |
| sm | 8px (`gap-2`, `p-2`) | Compact row padding, badge vertical padding |
| md | 16px (`px-4`, `gap-4`) | Default element spacing, form field padding |
| lg | 24px (`p-6`, `gap-6`) | Section padding, stat-box padding |
| xl | 32px (`gap-8`) | Layout gaps between Wizard stages |
| 2xl | 48px | Major section breaks (stage separator) |
| 3xl | 64px | Page-level top padding |

### Locked Pre-Phase Atoms — Exempt from Spacing Scale

The following values are inherited from existing locked atoms and are **exempt from the 8px-grid rule**. They must not be changed in this phase.

| Value | Source atom | Why exempt |
|-------|-------------|-----------|

## Typography

Consolidated from `tailwind.config.js` `fontSize` block. Maximum 4 sizes declared.

| Role | Tailwind class | Size | Weight | Line Height | Usage in this phase |
|------|---------------|------|--------|-------------|---------------------|
| Small / Label | `text-small` | 12px | 500 | 16px | Stage indicator labels, badge caps (uppercase), stat-box unit labels, stat-box sub-labels, group section subtitles |
| Body | `text-body` | 14px | 400 | 20px | Table rows, drift entry details, reason text, suggested-action hints |
| Section heading | `text-lg` | 16px | 600 | 22px | Page title "Datenmigration / Cutover", stage heading ("Profile", "Dry-Run", "Commit"), Drift-Group-Header person name |
| Display / Hero | `text-h1` | 22px | 600 | 28px | Stat-box primary number (quarantine count, row count) |

Active weights: 400 (body) and 600 (semibold — labels, headings, badges).

Body baseline: 16px on `html/body` (from `input.css`), overridden to 14px (`text-body`) inside table rows and form fields per existing project pattern.

**Consolidation notes (applied in this revision):**
- The former `text-micro` (11px) role is merged into `text-small` (12px). All usages previously documented as `text-micro` use `text-small` instead.
- The former `text-h2` (18px) page-heading role is merged into `text-lg` (16px). The page title "Datenmigration / Cutover" uses `text-lg font-semibold`.

**Primary focal point:** The `StageIndicator` at the top of the content panel (communicates wizard position) combined with the active stage's primary stat box `text-h1` value (quarantine count or row count) — these two elements carry the user's attention at every stage transition.

  - Active: `bg-accent text-accent-ink border-accent`
  - Complete: `bg-good text-white border-good` + checkmark glyph
  - Inactive: `bg-surface text-ink-muted border-border`

### Drift Entry Row Columns

| Column | Width | Content |
|--------|-------|---------|
| Date | `w-36` | ISO date + `text-ink-muted` weekday code, e.g. "2026-05-08 (Fri)" |
| Hours | `w-20 text-right` | `text-body font-mono text-ink` amount |

## States

### Loading State (D-09)

During Dry-Run re-run after a resolve action:
- Action button that triggered: `opacity-50 cursor-not-allowed` + inline `"..."` text replacement
- Drift list overlay: `pointer-events-none opacity-60` on the `DryRunStage` wrapper only
- Spinner: Unicode in `font-mono` rotating via inline `animation: spin 1s linear infinite` on a wrapper span

### Empty Quarantine List (Dry-Run clean)

When `quarantined_rows == 0` in the Dry-Run result:
- Replace drift list with a centered `bg-good-soft border border-good rounded-lg p-6 text-center`
- Heading: `text-lg text-good` — "Keine offenen Drifts"
- Body: `text-body text-ink-muted` — "Alle Einträge können automatisch migriert werden. Weiter zum Commit-Schritt."
- Commit button in Stage 3 becomes active

### Idempotenz Banner (post-commit re-open)

Shown when `absence_range_source_active == Some(true)` (Plan 08-07 `FeatureFlagsState`):
- `bg-accent-soft border-l-4 border-accent rounded-md px-4 py-3 flex items-start gap-3 mb-4`
- Icon: `i` in `text-accent font-mono`
- Text: `text-small text-accent font-semibold` heading + `text-body text-ink-soft` body

### Error Toast

On API error (non-200 from any cutover endpoint):
- Existing project pattern: `bg-bad-soft border border-bad rounded-md px-4 py-3 text-body text-bad`
- Displayed inline at top of stage content panel
- Not a full-page error — stage remains interactive

## Interaction Contracts

### Stage Progression

- Stage 1 to 2: Active after Profile API call succeeds (non-empty result). "Weiter" disabled if Profile not yet loaded.
- Stage 2 to 3: Active only when last Dry-Run result has `quarantined_rows == 0`.
- Stage 3 Commit button: Active only when Type-to-confirm input equals "CUTOVER" exactly (case-sensitive) AND user has `cutover_admin` privilege (D-23).
- Back navigation: Always available for Stage 2 to 1 and Stage 3 to 2. After commit success, all navigation is disabled (stages become read-only).

### Auto-Re-Run (D-08)

Triggered by: Convert (single), Delete, Bulk-Convert. NOT triggered by Skip (frontend-only).
- Implementation: increment `CUTOVER_DRIFT_REFRESH: GlobalSignal<u64>` (analog `ABSENCE_REFRESH` in Phase 8)
- While re-run is in flight: action button replaced with `"..."`, drift list rendered with `opacity-60 pointer-events-none`
- On success: drift list updates from new `refreshed_drift_report` inline response (if backend provides it) or separate `gate-dry-run` fetch
- On error: error toast shown inline; list reverts to previous state

### Skip Action (D-05)

- Instantly removes row from local UI list via `Vec::retain` on the local signal
- No backend call, no confirmation
- After page reload the row reappears (no persistence)

### Type-to-confirm Input

- `TextInput` with `input_type="text"` placeholder "CUTOVER"
- `Btn(Danger)` "Commit Cutover" has `disabled={input_value != "CUTOVER"}`
- Input renders with `border-bad` focus ring when non-empty and non-matching, normal `--accent` focus ring when empty or matching

### Privilege Gate (D-23)

HR users: Stage 1 + 2 fully functional. Stage 3 visible but Commit button rendered as `Btn(Danger)` with `disabled=true` + tooltip key `CutoverPrivilegeStage3`.
`cutover_admin` users: full Stage 3 access.
Neither: page hidden at router level (or 403 error state).
