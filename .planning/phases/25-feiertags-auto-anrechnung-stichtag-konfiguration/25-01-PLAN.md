<objective>
Extend the existing Phase-24 toggle infrastructure with a nullable `value TEXT` column so the
holiday-automatic activation date (Stichtag) can be persisted as an ISO date string, and expose
read/write of that value through the full DB -> DAO -> Service -> REST stack. This is the
foundation consumed by the reporting derive-on-read plan (25-02, reads the cutoff) and the
frontend settings field (25-03, writes the cutoff).

Purpose: HCFG-01 storage + HCFG-02 backend half. Implements D-25-04 (reuse toggle infra, add
value column) and D-25-05 (value is the authoritative gate; default = automation off).

Output: Migration + seed, `ToggleEntity.value` / `Toggle.value` / `ToggleTO.value`,
`get_toggle_value`/`set_toggle_value` at DAO + Service tiers, three new admin-gated REST
endpoints, and a regenerated `.sqlx` offline cache.
</objective>
