```rust
// Phase 4 (Plan 04-06) — D-Phase4-09: deprecated POST /extra-hours
// categories return 403 with a snapshot-locked JSON body shape so the
// frontend can branch on `error == "extra_hours_category_deprecated"`.
Err(RestError::ServiceError(ServiceError::ExtraHoursCategoryDeprecated(category))) => {
    let body = rest_types::ExtraHoursCategoryDeprecatedErrorTO {
        error: "extra_hours_category_deprecated".to_string(),
        category: format!("{:?}", category).to_lowercase(),
        message: "Use POST /absence-period for this category".to_string(),
    };
    Response::builder()
        .status(403)
        .header("Content-Type", "application/json")
        .body(Body::new(serde_json::to_string(&body).unwrap_or_default()))
        .unwrap()
}
```

## Decisions Made

All key decisions are listed in the frontmatter `key-decisions` block. Highlights:

1. **Service-layer auth (T-04-06-01 mitigation):** REST handlers do NOT pre-check permissions. CutoverService::run/profile gate internally — single enforcement point catches both REST and non-REST callers.
2. **Snapshot-locked 403 body shape:** `{error, category, message}` is now part of the OpenAPI surface; future changes to the body shape will fire the insta-snapshot diff in CI.
3. **DI re-order MANDATORY:** FeatureFlagServiceImpl Arc::new MUST sit before ExtraHoursServiceImpl Arc::new (Plan 04-04 hand-off). Verified by awk-pattern in acceptance criteria; lines 824 and 829 respectively.
4. **Empty-body request DTOs as forward-compat strategy:** future field additions to gate-dry-run/commit/profile request bodies are explicit-review-gated by the OpenAPI snapshot diff; non-Option fields are snapshot-breaking.
5. **Marker-struct + assoc-type-alias DI pattern verbatim from AbsenceServiceDependencies:** zero struct fields, no derive Clone; the gen_service_impl!-generated *ServiceImpl<Deps> holds all Arcs.
