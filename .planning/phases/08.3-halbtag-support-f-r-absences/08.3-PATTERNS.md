Für `DayFractionTO` ist die DB-Repräsentation lowercase (`'full'`/`'half'`); falls Wire-Symmetrie mit DB gewünscht: `#[serde(rename_all = "snake_case")]`. **Plan-Phase entscheidet** — beide Varianten kompilieren.

## Shared Patterns

### Transaction Management

**Source:** `service_impl/src/absence.rs:392-396, 511-512`
**Apply to:** Alle erweiterten Service-Methoden (`AbsenceServiceImpl::create/update`, `CutoverServiceImpl::convert_quarantine_entry/bulk_convert_quarantine_rows`)

```rust
let tx = self.transaction_dao.use_transaction(tx).await?;
// ... DAO calls mit tx.clone() ...
self.transaction_dao.commit(tx).await?;
```

Pattern bleibt unverändert — `day_fraction` ist keine Cross-Tx-Operation.

### Permission-Gates

**Source:** `service_impl/src/absence.rs:164-173` (`hr.or(sp)` für HR ∨ self)
**Apply to:** `AbsenceServiceImpl::create/update` — Halbtag-Eingabe folgt den selben Permission-Regeln wie Range-Eingabe.

```rust
let (hr, sp) = join!(
    self.permission_service
        .check_permission(HR_PRIVILEGE, context.clone()),
    self.sales_person_service.verify_user_is_sales_person(
        request.sales_person_id,
        context,
        tx.clone().into()
    ),
);
hr.or(sp)?;
```

`CutoverServiceImpl::convert_quarantine_entry` nutzt `CUTOVER_ADMIN_PRIVILEGE`-Gate (Z. 421-423) — Halbtag-Eingabe ist Teil der Cutover-Action und erbt diese Berechtigung.

### Error-Handling

**Source:** `rest/src/cutover.rs:194-241` (`error_handler` + `ServiceError::ValidationError` → HTTP 422)
**Apply to:** Alle erweiterten REST-Handler. `day_fraction` parsing-Errors müssen via `ValidationError` (nicht `panic!`) propagiert werden — Pattern wie `manual_range`-Parsing (Z. 202-228).

### `gen_service_impl!`-Konvention

**Source:** `service_impl/src/billing_period_report.rs:39-51`
**Apply to:** Keine neuen Service-Constructions. `AbsenceServiceImpl` und `CutoverServiceImpl` bleiben in der bestehenden Form — `day_fraction` ist Daten-Payload, kein neuer Dep.

### OpenAPI-Annotations

**Source:** `rest/src/cutover.rs:177-189` (`#[utoipa::path(...)]`)
**Apply to:** Bestehende Handler — keine Annotation-Changes nötig. `DayFractionTO` ist transitiv via `AbsencePeriodTO` / Request-DTOs eingebunden.

### i18n FUI-A-09 Compliance

**Source:** `shifty-dioxus/src/i18n/mod.rs:680-755` (Absence-Block) + Z. 775-897 (Cutover-Block)
