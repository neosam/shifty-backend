# Phase 8.3 — UI Design Contract

> Visual and interaction contract for Phase 8.3: Halbtag-Support für Absences.
> Diese Phase fügt EIN neues Form-Feld (`day_fraction: Full | Half`) zu drei
> bereits gebauten Surfaces hinzu:
>
> 1. `AbsenceModal` (`shifty-dioxus/src/page/absences.rs`) — Phase 8 Baseline.
> 2. `DriftEntryRow` + `DriftGroupSection` (`shifty-dioxus/src/page/cutover_admin.rs`) — Phase 8.1 Baseline.
> 3. `ManualConvertModal` (im selben `cutover_admin.rs`) — Phase 8.2 Baseline.
>
> Es werden **keine** neuen Design-Tokens (Farben, Spacings, Typografie-Skalen)
> eingeführt. Das gesamte Design-System aus Phase 8 + 8.1 bleibt gelockt und
> wird 1:1 konsumiert. Diese SPEC dokumentiert **nur die Erweiterungs-Slots**
> sowie die i18n-Coverage für den neuen Halb/Ganz-Mechanismus.
>
> Erstellt von gsd-ui-researcher, verifiziert von gsd-ui-checker.

## Interaction Contract

### AbsenceModal — `day_fraction`-Eingabe

**Verhalten:**
- Default beim Öffnen im Create-Mode: `Full`. Begründung: Halbtage sind der
  Sonderfall (Heiligabend/Silvester); Vollurlaub ist der Normalfall.
- Default beim Öffnen im Edit-Mode: aus `editing_period.day_fraction` befüllt.
- Bei Range > 1 Tag und `day_fraction = Half` → erlaubt, keine Validierung.
  CONTEXT.md D-04-default: gilt einheitlich für alle Tage der Range (Sonderfall
  selten, für Heiligabend/Silvester ist Range = 1 Tag).
- Hint-Text unter dem Feld wechselt reaktiv: `Full` zeigt `AbsenceFieldDayFractionFullHint`,
  `Half` zeigt `AbsenceFieldDayFractionHalfHint`.
- Submit-Verhalten unverändert — POST/PUT `AbsencePeriodTO` enthält jetzt
  `day_fraction: DayFractionTO`.

**Layout in `AbsenceModal` Body-Grid:**

```
grid grid-cols-2 gap-3:
  [VersionConflictBanner — col-span-2, conditional]
  [Field Mitarbeiter — col-span-2]
  [Field Kategorie — col-span-2]
  [Field Von]                       [Field Bis (mit error-Slot)]
  [Field Tageshälfte — NEU, span-2 falls SelectInput; alternativ ein-spaltig falls Plan-Phase eine Zwei-Felder-Zeile wählt]
  [SelfOverlapBanner — col-span-2, conditional]
  [Field Beschreibung — col-span-2]
  [Vorschau-Box — col-span-2, bg-surface-alt]
  [WarningList — col-span-2, conditional nach Save]
```

**Default-Empfehlung an Plan-Phase:** `Field` mit `span: Some(2u8)` (full-width)
und `SelectInput` darin — analog Kategorie-Feld (Z. 1031-1061 in `absences.rs`).
Begründung: konsistente Visual-Sprache. Alternative (`FormCheckbox` `Halber Tag`)
ist kürzer aber bricht die "jede Daten-Eigenschaft ist ein Field+SelectInput"-
Konsistenz mit Kategorie. Plan-Phase entscheidet final.

### DriftEntryRow — `day_fraction`-Toggle

**Verhalten:**
- Default: `Full`.
- **Optional** (Plan-Phase D-08.3-BE-02): Auto-Vorschlag aus Heuristik —
  wenn `(entry.0.amount - target_contract_hours * 0.5).abs() < CONTRACT_HOURS_EPSILON`,
  Default vorausgewählt auf `Half`. Sichtbar als Hint `CutoverDriftHalfDaySuggestion`
  über der Row (oder als Tooltip am Toggle).
- Convert-Button dispatcht `CutoverAction::ConvertSingle { extra_hours_id, day_fraction }`
  (Erweiterung der existing Action). Bulk-Convert analog falls D-08.3-UI-05
  gruppenweite Auswahl bestätigt.

**Layout:**

```
grid grid-cols-12 gap-2 (existing layout, Z. 726 in cutover_admin.rs):
  [col-span-3: Date + Amount]
  [col-span-3: Reason badge]
  [col-span-2: suggested_action]
  [col-span-4: Actions: ⟨day_fraction toggle⟩ Convert Edit Delete Skip]
                       ↑ NEU
```

**Empfehlung an Plan-Phase:** Der Toggle ist ein **kompakter `SelectInput`**
mit `text-small px-2 py-1`-Override, **vor** den vier Action-Buttons im
`col-span-4 flex gap-1 justify-end`-Bereich. Falls die Action-Spalte zu eng
wird (>= 5 Elemente), kann der Toggle in einen separaten Sub-Container
`col-span-2 / col-span-2`-Split (Toggle separat, Actions separat) — Plan-Phase
entscheidet.

### ManualConvertModal — `day_fraction`-Eingabe

## Accessibility Contract

- **AbsenceModal-Erweiterung:** Neues `Field` mit `<label>`-Wrapper (semantisch
  korrekt via `Field`-Atom). `SelectInput` (falls gewählt) ist ein nativer
  `<select>` mit `<option>`-Kindern — Tastatur-/Screen-Reader-Support
  inhärent. `FormCheckbox`-Alternative bringt natives `<input type="checkbox">`
  mit `<label>`-Wrapper (auch inhärent zugänglich).
- **DriftEntryRow-Toggle:** Der kompakte Per-Row-Select braucht ein
  `aria-label` (i18n-Key `CutoverDriftDayFractionAria`), weil das visuelle
  Label fehlt (Spalten-Header-Pattern Plan-Phase-D-08.3-UI-05).
