## Plan-Phase decisions

- **D-08.3-FE-01** (resolves CONTEXT OQ-3 / UI-SPEC.md D-08.3-UI-04 / Frontend-UI-Pattern): `SelectInput` (Full/Half) für alle drei Surfaces — `AbsenceModal`, `DriftEntryRow`, `ManualConvertModal`. Begründung: UX-Konsistenz mit dem existierenden Kategorie-Feld in `AbsenceModal`, semantisch klarer als Checkbox (2 explizite Optionen statt boolean), gleiche Spacing-Tokens. `FormCheckbox`-Alternative wurde verworfen (würde Field+SelectInput-Konsistenz brechen). UI-SPEC.md dokumentiert beide als zulässig — Plan-Phase wählt SelectInput.

- **D-08.3-FE-02** (resolves CONTEXT OQ-2 / UI-SPEC.md D-08.3-UI-06 / Auto-Vorschlag-Hint sichtbar): Frontend rendert einen sichtbaren `CutoverDriftHalfDaySuggestion`-Hint innerhalb der `DriftEntryRow`, wenn die ~4h-Heuristik anschlägt (`(entry.amount - contract_hours_per_day * 0.5).abs() < 0.5`). Operator entscheidet weiterhin manuell (Toggle bleibt auf `Full`-Default — Plan-Phase NICHT auto-flipped, damit Operator den Hint bewusst quittiert). Backend bleibt passiv (D-08.3-06 in Plan 05). Verifiziert durch 2 zusätzliche SSR-Snapshot-Tests in Task 2: `drift_entry_row_renders_half_day_suggestion_hint_on_4h_amount` (positiver Case, Hint vorhanden) und `drift_entry_row_omits_half_day_suggestion_hint_for_full_day_amount` (negativer Case, Hint fehlt bei 8h Vacation). Begründung: Operator-Klarheit — der Hint macht sichtbar, warum Half wahrscheinlich richtig ist; ohne Hint wäre die Heuristik unsichtbar für den Operator (Surface-Disconnect).

- **D-08.3-FE-03** (UI-SPEC.md D-08.3-UI-05 / Bulk-Toggle): KEIN gruppen-weiter Halb/Ganz-Toggle in `DriftGroupSection`. `BulkConvert`-Dispatch verwendet hardcoded `day_fraction: DayFractionTO::Full`. Operator nutzt Per-Row-Convert für Halbtag-Einträge (Halbtag-Bulk-Use-Case ist selten — Heiligabend ist typischerweise EIN Tag pro Person). Der i18n-Key `CutoverBulkDayFractionLabel` bleibt deklariert + übersetzt (Plan 01 D-08.3-02 + Plan 03) für Future-Reservation; ist aber in dieser Phase NICHT konsumiert. Begründung: weniger UI-Komplexität; Single-Point-Change für zukünftige Erweiterung; reduzierte Test-Oberfläche. Falls eine spätere Phase den Toggle braucht, ist es ein lokalisierter Single-Point-Patch im Group-Header von `DriftGroupSection`.
