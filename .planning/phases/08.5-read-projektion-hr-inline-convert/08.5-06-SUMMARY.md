## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] rsx!-Makro erlaubt keine let-Bindungen in if-let-Bloecken**
- **Found during:** Task 3 (Page-Verdrahtung)
- **Issue:** Der Plan beschreibt die Modal-Verdrahtung als Inline-Code innerhalb von `rsx! { if let Some(m) = ... { let modal_category = ...; AbsenceConvertModal { ... } } }`. Dioxus erlaubt keine `let`-Bindungen in `if`-Bloecken innerhalb von `rsx!`.
- **Fix:** Modal-Elemente als Rust-Variablen (`convert_modal_element`, `edit_hours_modal_element`) vor dem `rsx!`-Aufruf berechnet. Im `rsx!`-Block nur noch `{convert_modal_element}` und `{edit_hours_modal_element}` referenziert.
- **Files modified:** shifty-dioxus/src/page/absences.rs

### Anmerkungen

**1. TDD RED/GREEN: HourlyMarkerRow bereits in Task 2 implementiert**
Plan beschreibt Task 3 als TDD. Da `HourlyMarkerRow` bereits in Task 2 vollstaendig implementiert wurde, waren die 3 SSR-Tests bei ihrer Erstellung direkt gruen (kein echter RED-Zustand). Die TDD-Commits wurden trotzdem korrekt als `test(...)` und `feat(...)` aufgeteilt, um die Commit-Historie nachvollziehbar zu halten.

**2. bump_absence_refresh() pub(crate) ausreichend**
`bump_absence_refresh()` ist `pub(crate)` in `service::absence`. Der direkte Aufruf aus `page::absences` (selbe Crate) funktioniert ohne Visibility-Aenderung.
