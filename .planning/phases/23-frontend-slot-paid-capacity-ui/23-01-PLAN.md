<objective>
Extend the week-view slot cell background coloring so a slot whose live paid-booking
count exceeds its paid-employee limit is tinted red (`bg-bad-soft`), distinct from the
orange understaffing tint, with the paid-overage state taking priority over
understaffing, visible to all roles.

Purpose: Make paid-capacity overruns immediately visible in the schedule (D-23-03/04/05).
Backend already delivers `max_paid_employees` and `current_paid_count` on the week-view
`Slot` state — this plan only changes the class-derivation logic and adds tests.

Output: Modified `cell_background_class` (3-arg), updated call site, 4 updated existing
unit tests + 3 new unit tests + 2 new SSR tests, all green; WASM build + clippy clean.
</objective>

        ""

}
