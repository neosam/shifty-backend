## Plan-Phase decisions

- **D-08.3-01** (resolves CONTEXT OQ-1 / Datenmodell-Form): Rust-Enum `DayFraction { Full, Half }` (type-safe, idiomatisch) + DB-Spalte `TEXT NOT NULL DEFAULT 'full'` mit `CHECK (day_fraction IN ('full', 'half'))`. Begründung: konsistent mit dem bestehenden `AbsenceCategoryEntity`-Pattern (Enum + TEXT, kein serde-rename_all); self-dokumentierend in DB-Dumps; CHECK-Constraint als defense-in-depth gegen DAO-Bugs. Alternative (INTEGER 0/1 oder REAL 0.5/1.0) verworfen wegen fehlender Self-Doc und Inkonsistenz mit Bestand. `Default = Full` auf Domain- und DTO-Ebene garantiert die no-drift-Migration für Bestandsdaten (CONTEXT.md LOCKED-Garantie).

- **D-08.3-02** (resolves CONTEXT OQ-6 / i18n-Key-Naming): Naming-Convention `Absence<Bereich><Bezeichnung>` und `Cutover<Bereich><Bezeichnung>` gemäß UI-SPEC.md Copywriting-Tabelle (Z. 150-178). Bodies in allen drei Locales (De/En/Cs) im selben Commit (FUI-A-09). Per-Locale-Reference-Matcher-Tests in Plan 03 (Pitfall-2-Guard aus Plan 08-04 D-26).
