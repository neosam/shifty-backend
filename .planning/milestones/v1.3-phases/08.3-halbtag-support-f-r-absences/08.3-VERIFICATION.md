## Human Verification

Folgende Aspekte sind durch automatisierte Tests gepinnt, profitieren aber von
einer kurzen manuellen UAT vor dem finalen Cutover-Switch (Plan 08.1-12):

1. **AbsenceModal — visuelle Sanity:**
   - Create-Mode: Halbtag-Feld zwischen Kategorie und Von/Bis sichtbar, Default
     `Ganztag` selektiert, Hint-Zeile wechselt reaktiv zwischen Full-Hint und
     Half-Hint beim Toggle.
   - Edit-Mode mit existing Halbtag-Absence: Half ist vorausgewählt.
2. **Cutover-Admin — Heiligabend-Pattern end-to-end:**
   - Operator sieht für einen 4h-Vacation-Drift-Eintrag den
     `CutoverDriftHalfDaySuggestion`-Hint unter der Action-Zeile.
   - Operator schaltet das Per-Row-Select auf `Half`, drückt Convert → die
     resultierende `absence_period`-Row hat `day_fraction = 'half'` in der DB.
   - Reporting für die betroffene Person halbiert die Vacation-Stunden im
     `BillingPeriodReport` entsprechend.
3. **Booking-Conflict-Suppression — UI:**
   - Halbtag-Vacation am 24.12. + ein Booking am selben Tag erzeugen keine
     Warning im Shift-Plan-Edit-Flow. Voll-Tag-Vacation erzeugt weiterhin die
     gewohnte Warning.
4. **ManualConvertModal (Karin-Pattern):**
   - Date-Range frei wählbar + Halb/Ganz-Select sichtbar, Default `Ganztag`.
   - Submit threaded den Wert; nach Convert: AbsencePeriod im Backend hat das
     gewählte `day_fraction`.
5. **Dioxus dev-server Smoke-Test:**
   - `dx serve --hot-reload` startet ohne Errors.
   - i18n-Locale-Switch zeigt korrekte Bodies für alle drei Sprachen
     (insbesondere `Tageshälfte` / `Day fraction` / `Část dne`).
6. **Bulk-Convert UX:**
   - `DriftGroupSection` rendert KEINEN gruppen-weiten Toggle (per D-08.3-FE-03).
     `BulkConvert` produziert Full-Day-AbsencePeriods. Für Halbtag-Migrations
     nutzt der Operator das Per-Row-Convert-Flow. Wenn der Operator das anders
     erwartet, ist es ein UI-Reservation-Hinweis (Future-Phase).
