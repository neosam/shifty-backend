## Plan-Phase decisions

- **D-08.3-04** (resolves CONTEXT OQ-4 / Range-Verhalten bei Half): Einheitliche Halbierung für ALLE Tage einer `AbsencePeriod`-Range mit `day_fraction = Half`. Kein per-Tag-Override, kein Zwei-Felder-Modell (Start/End-Day-Fraction). Begründung: AbsencePeriod hat genau ein `day_fraction`-Feld; klassischer Anwendungsfall (Heiligabend/Silvester) ist Range = 1 Tag; Range > 1 Tag mit Halbtag ist legitimer aber seltener Sonderfall (z. B. eine ganze Woche reduziert auf Halbtage). Verifiziert durch Test `test_derive_hours_half_day_two_day_range_uniform_halving` (jeder Tag der 2-Tage-Range wird halbiert, Summe = 1 voller Vertragstag).

- **D-08.3-05** (resolves CONTEXT OQ-5 / Konflikt-Warning): KEIN neuer `WarningTO`-Variant. Der bestehende `Warning::BookingOnAbsenceDay`-Emitter in `service_impl/src/shiftplan_edit.rs` Z. 482-489 wird ergänzt um einen `day_fraction == Full`-Filter; bei `Half` wird die Warning unterdrückt ("schweigend toleriert" per CONTEXT.md `<domain>` "Liefert NICHT" + CONTEXT.md `<deferred>`).
