## Decisions Made

### D-1 (Plan 10): No visibility bump for heuristic helpers

`detect_weekly_lump_sum`, `iso_week_range`, and `lookup_active_contract` are file-private in `service_impl/src/cutover.rs`. The diagnose tests drive the heuristic through `service.run(true, ...)` and observe `total_clusters` / `quarantined_rows`, mirroring all existing Plan-08-09 lump-sum tests. This avoids `pub(crate)`-leakage for diagnose-only purposes and keeps the test signal aligned with the live migration path.

### D-2 (Plan 10): No code fix for the Karin pattern (bleibender gap)

A "guess which weekly total the user intended" patch would silently produce semantically wrong absence_periods — e.g. accept 40h but store an absence_period whose `derive_hours_for_range` yields 36h, creating permanent drift downstream. The Plan-08-09 D-19 directive explicitly says: if cause is design-level, document as gap and route via operator-resolution. Edit-modal in the 8.1-UI lets the operator correct the amount per row before Convert. **gap-1 (a) is the deliberate, persistent boundary of the auto-heuristic.**

### D-3 (Plan 10): Tests assert observed, not ideal, behaviour

Each diagnose test asserts the heuristic's actual output (e.g. for Karin: `total_clusters = 0`, `quarantined_rows = 1`). Future heuristic refinements that change the verdict will update both the assertion and the test's doc-comment in the same commit, keeping the test as a living-document of the heuristic's coverage. No `#[ignore]` markers are needed — all three tests assert outcomes the heuristic IS supposed to produce today.
