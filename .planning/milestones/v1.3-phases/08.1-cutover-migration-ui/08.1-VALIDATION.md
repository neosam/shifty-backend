> Per-phase validation contract for feedback sampling during execution.
> See `08.1-RESEARCH.md` `## Validation Architecture` section for the full Phase Requirements → Test Map.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Backend: `cargo test` (workspace, mockall + in-memory SQLite); Frontend: `cargo test -p shifty-dioxus` (dioxus-ssr snapshots + per-locale i18n matchers); WASM: `cargo build --target wasm32-unknown-unknown` from `shifty-dioxus/` |
| **Config file** | `Cargo.toml` (root workspace) + `shifty-dioxus/Cargo.toml`; no extra runner config |
| **Quick run command** | `cargo test --workspace --no-fail-fast --quiet 2>&1 \| tail -50` (root) |
| **Full suite command** | `nix develop -c cargo test --workspace && (cd shifty-dioxus && nix develop -c cargo test && nix develop -c cargo build --target wasm32-unknown-unknown)` |
| **Estimated runtime** | Backend workspace ~60-90s; Frontend tests ~20-30s; WASM build ~30-60s warm / ~3-5min cold |

---

## Sampling Rate

- **After every task commit:** Run `cargo test --workspace --quiet` (or `cargo test -p <crate>` for crate-local edits)
- **After every plan wave:** Full suite (workspace + frontend + WASM-build-gate)
- **Before `/gsd-verify-work`:** Full suite must be green; WASM build green; `cargo clippy --workspace -- -D warnings` clean
- **Max feedback latency:** ~90s for backend quick run; ~3min for frontend incl. WASM cold
