# Requirements — v2.3: PDF-Export — Browser-Look & Download-Button

## Kontext

Der in v2.2 gelieferte Nextcloud-PDF-Export via WebDAV (Phase 48) rendert
PDFs, die praktisch unlesbar sind: ein starres mm-Absolut-Layout, keine
sichtbaren Slot-Zellen, keine Uhrzeiten je Slot, nur Sales-Person-Namen in
Zeilen und Wochentage in Spalten. Das WebDAV-Ablegen funktioniert
technisch, hilft dem Team aber nicht.

v2.3 löst zwei Probleme zusammen:

1. Die PDFs müssen visuell dem Schichtplan im Browser
   (`shifty-dioxus/src/page/shiftplan.rs`) ähneln — Slots als Zellen mit
   Uhrzeiten, Bookings mit Sales-Person-Name, so dass ein A4-Ausdruck ohne
   externe Referenz nutzbar ist. Zusätzlich soll jede Seite den
   Renderzeitpunkt tragen, damit Ausdrucke datierbar sind.
2. Auf der Schichtplan-Seite selbst gibt es einen Download-Button für die
   im UI selektierte Kalenderwoche, damit man den Export ad-hoc nutzen kann
   und nicht auf den täglichen Cron-Slot warten muss.

Der Reihenfolge-Trick: Phase 49 liefert nur den Button (nutzt den alten,
unlesbaren Renderer), Phase 50 tauscht den Renderer aus. So kann das
Rendering-Ergebnis in Phase 50 direkt per Button-Klick verifiziert werden.

## Nicht-Ziele

- **i18n der PDF-Texte** — bleibt Deutsch (Datumsformat, „Schichtplan KW"-
  Header, Timestamp-Label). Frontend-Button-Label wird i18nt.
- **Byte-Determinismus des Renderers** — fällt implizit durch PDF-02
  (Renderzeitpunkt). Der v2.2-Determinismus-Vertrag (fixe Metadata
  2000-01-01) wird in Phase 50 aufgehoben. WebDAV-Overwrite bleibt korrekt.
- **Neue Cargo-Deps** — keine erwartet. `printpdf` bleibt.
- **Migration** — keine. Snapshot-Schema-Version bleibt **12**.

## Requirements

### PDF-01 — PDF-Layout wie Browser-Wochenansicht

- [x] Das gerenderte PDF entspricht sichtbar dem Schichtplan im Browser:
  Landscape A4, sieben Wochentag-Spalten, Slots als eigene Zellen mit
  Uhrzeit-Label, Sales-Person-Namen in der Zelle, „Schichtplan KW {NN}
  ({JJJJ})"-Kopfzeile.

**Ausgeliefert in:** Phase 50 (kompletter Renderer-Rewrite mit
Hybrid-Stack-Layout, `add_rect` + `PaintMode::Stroke` für Slot-Rahmen,
dynamischer Sonntag-Spalte).

**Verifikation:** Backend-Test rendert Beispieltag mit Slots + Bookings,
prüft dass alle relevanten Strings im PDF-Textstream vorkommen. Human UAT
D-50-17 2026-07-04 bestätigt.

**Design-Deviation post-UAT:** `+ N weitere` Overflow-Marker (D-50-03/04)
und `(freiwillig)`-Suffix (D-50-06/07) auf User-Wunsch nach UAT entfernt —
Boxen wachsen mit, Namen komma-separiert, paid/unpaid irrelevant im PDF.

### PDF-02 — Renderzeitpunkt auf jeder Seite

- [x] Jede PDF-Seite trägt den Renderzeitpunkt sichtbar, Format „Erstellt am
  DD.MM.YYYY HH:MM Uhr" (lokale Zeit des Backend-Servers).

**Ausgeliefert in:** Phase 50 — Header-Timestamp oben-rechts (D-50-09).
`resolve_render_timestamp()` mit `now_local()` + UTC-Fallback +
`tracing::warn!`-Log bei `IndeterminateOffset` (D-50-12).

**Verifikation:** Backend-Test injiziert `FIXED_RENDER_TIMESTAMP` über den
Renderer-Parameter, prüft dass der Renderzeitpunkt im Textstream steht.
Service-Test `now_local_fallback_to_utc_on_indeterminate_offset` grün.

### PDF-03 — Download-Button auf Schichtplan-Seite

- [x] Auf `shifty-dioxus/src/page/shiftplan.rs` gibt es einen Download-Button
  neben dem iCal-Button. Klick lädt das PDF der aktuell im UI selektierten
  Kalenderwoche.

**Ausgeliefert in:** Phase 49 — `<a>`-Anchor mit Cookie-Auth, Dateiname
`schichtplan-{JJJJ}-KW{NN}.pdf` via Content-Disposition, i18n-Key
`PdfDownload` in de/en/cs.

**Verifikation:** Manueller UAT — Klick, Datei landet mit korrektem
Dateinamen im Browser-Download.

### PDF-04 — Sichtbarkeits-/Download-Gate auf `week_status`

- [x] Button nur sichtbar bei `WeekStatus ∈ {Planned, Locked}`; Backend gibt
  bei `{Unset, InPlanning}` HTTP 409 zurück (Defense-in-Depth).

**Ausgeliefert in:** Phase 49 — pure Predicate-Fn
`should_show_pdf_button(status, shiftplan_id) -> bool` mit 8-case
Test-Matrix; Handler-level 409 pre-check via `WeekStatusService`; JSON
error body mit stabilem Code `week-not-releasable` (D-49-03).

**Verifikation:** Backend-Integrationstest gegen alle vier Status-Werte;
FE cargo-Test auf pure Predikat-Fn (8 Fälle).

### PDF-05 — Kein Admin-Gate auf Download

- [x] Download für jeden authentifizierten User erreichbar (Employee
  eingeschlossen). Keine Sales-Person-Filterung.

**Ausgeliefert in:** Phase 49 — Handler ohne Admin-Gate; Employee-Auth
liefert 200.

**Verifikation:** Backend-Integrationstest mit Employee-Auth-Context
prüft 200 statt 403.

## Constraints (alle eingehalten)

- **Byte-Determinismus** bewusst gebrochen (Timestamp).
- **Renderer bleibt pure** — 5. Parameter `render_timestamp: OffsetDateTime`
  (D-50-11); Aufrufer injizieren via `resolve_render_timestamp()`.
- **WebDAV-Scheduler** nicht angefasst — nutzt neuen Renderer automatisch
  via `PdfShiftplanService`-Delegation (Phase 49 D-49-08).
- **Kein Snapshot-Bump, keine Migration, keine neue Cargo-Dep** — nur
  `local-offset`-Feature auf existierendem `time`-Crate.

## Post-Ship Hotfix

`fix(pdf-export): tolerate per-week ValidationError + fix cron seed to
6-field` — WebDAV-Scheduler ignoriert einzelne Wochen mit ValidationError
statt komplett zu fallen; Cron-Seed-Format korrigiert. Gerollt als v2.3.1
(Commit `754f94f`).
