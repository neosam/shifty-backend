# Requirements: Milestone v2.2 — Aufräumen, WebDAV-Export & Wochentag-Muster

**Charakter:** Aufräum-Milestone plus zwei substanzielle Features (PDF/WebDAV-Export, Wochentag-Anwesenheits-Muster). Arbeitet die verbliebenen v1.x-Carry-over-Bugs, den v1.11-Dioxus-Warnings-Rest und den v2.1-Reporting-Fehlschlag ab, ergänzt REST-Hygiene und den ursprünglich für v2.2 vorgesehenen Nextcloud-PDF-Export.

**Rahmen:**

- **Snapshot-Schema-Version:** bleibt **12** — kein Bump; Wochentag-Metrik ist reines Read-Aggregat (analog v2.1 AVG-01/02/03).
- **Migration:** keine erwartet (PDF/WebDAV-Config kommt über Umgebungsvariablen; keine neue Tabelle).
- **Neue Dependencies:** ein PDF-Renderer + WebDAV-Client — wird in discuss-phase 48 konkretisiert.
- **Phasen-Nummerierung:** setzt bei **43** fort.

## v2.2 Requirements

### Special-Days-Feintuning (SDF)

- [x] **SDF-03**: Der Special-Days-Loader in den Einstellungen lädt die Feiertage nach **Kalenderjahr** (`date.year()`), nicht nach ISO-Wochenjahr — behebt den 1.1.-Anzeige-Bug (`sd_year.set(iso_year)` in `settings.rs`).
- [x] **SDF-04**: Der Inline-Hinweis „already exists" im Special-Days-Anlegen-Card wird an das aktuelle Replace-Verhalten (in-place Ersetzung) angepasst — i18n-Copy in de/en/cs korrigiert. (Löst WR-02 v1.11.)
- [x] **SDF-05**: Beim Umstellen eines Tages im Schichtplan-Dropdown von „Feiertag" auf „Kurzer Tag" (bzw. umgekehrt) erscheint keine Fehlermeldung mehr; das in-place-Replace greift analog SDF-01.

### Frontend-Bugs (BUG)

- [x] **BUG-01**: `save_slot_edit` in `shifty-dioxus` hält den `SLOT_EDIT_STORE`-Write-Borrow **nicht mehr über `.await`** — Panic-Risiko behoben (WR-02 v1.10 aufgelöst).
- [x] **BUG-02**: `list_user_invitations`-Loader liefert bei Parse-Fehlern einen **sichtbaren** Fehler statt einer silent-empty-Fallback-Liste.
- [x] **BUG-03**: Verbleibende Modals, die die drag-safe `BackdropPress`-Logik noch nicht nutzen, sind darauf migriert; Text-Selektion im Panel schließt das Modal nicht mehr.

### Hygiene (HYG)

- [x] **HYG-03**: Der `shifty-dioxus`-Workspace kompiliert warnungsfrei mit `cargo build` UND `cargo clippy` (Fortsetzung HYG-01/02; ~198 pre-existing Lints abgearbeitet oder begründet `#[allow]`).
- [x] **HYG-04**: „Edit structure" (Schichtplan-UI) ist vollständig i18n-abgedeckt (de/en/cs).
- [x] **HYG-05**: Alle REST-Services liefern den korrekten `Content-Type` (JSON: `application/json`), verifiziert durch einen automatisierten Test-Layer, der jeden Endpoint gegen die erwartete Antwort-MIME prüft.

### Impersonation-Follow-Up (IMP)

- [x] **IMP-05**: Der Test `i18n_impersonation_keys_match_german_reference` ist grün — kanonische Copy-Entscheidung (Test-Referenz vs. De-Übersetzung „🥸 Agieren") festgelegt und umgesetzt. Pre-v2.1 Carry-over aus Phase 37-02 (Commit 83a0d91).

### Reporting-Update (RPT) — ersetzt v2.1-AVG

- [x] **RPT-01**: Im Mitarbeiter-Report (HR-Stats-Block, `employee_view.rs`) wird pro flexiblem Mitarbeiter (`is_dynamic == true`) für den angezeigten Report-Zeitraum eine **Wochentag-Zeile** angezeigt, die pro Wochentag die **Anzahl der Anwesenheitstage** UND den **Anteil in Prozent** (gegenüber gezählten Kalenderwochen im Zeitraum) zeigt — z.B. „Mo: 8 (80%) · Di: 3 (30%) · Mi: 7 (70%) · ...". „Anwesenheitstag" definiert wie in v2.1 (D-AVG-02/03): mindestens ein Tages-Eintrag der Kategorie `Shiftplan` / `ExtraWork` / `VolunteerWork` mit `hours > 0`.
- [x] **RPT-02**: Die v2.1-Kennzahl „Ø Std/Anwesenheitstag" (`average_hours_per_attendance_day`) wird durch RPT-01 **ersetzt** und aus dem FE-HR-Stats-Block entfernt; der Endpoint `/report/{id}/attendance-statistics` liefert die neue Wochentag-Struktur (kein Snapshot-Bump). HR-Gate + `is_dynamic`-Filter bleiben.
- [x] **RPT-03**: i18n de/en/cs für die neuen Wochentag-Labels, den Tooltip und den Leerzustand (< 1 Kalenderwoche im Zeitraum).

### PDF-Export nach Nextcloud (EXP)

- [x] **EXP-01**: Ein Backend-Task generiert **regelmäßig** (täglich, konfigurierbar) einen PDF-Export der Wochen-Schichtpläne (alle aktiven Sales-Persons; konfigurierbarer Wochen-Horizont, Default: aktuelle Woche + 1 Folgewoche) und lädt ihn per **WebDAV** in einen konfigurierbaren Ordner nach **Nextcloud** hoch.
- [x] **EXP-02**: Die WebDAV-Konfiguration (URL, User, App-Token, Zielordner, Wochen-Horizont, Cron-Zeitplan) ist über Umgebungsvariablen konfigurierbar; kein Klartext-Passwort im Code oder in der DB.
- [x] **EXP-03**: Der Export ist **admin-gated** aktivierbar/deaktivierbar (Feature-Flag/Config), meldet den letzten Erfolg/Fehler in einem Admin-Panel und wiederholt den Upload beim nächsten Lauf bei transienten WebDAV-Fehlern (keine dauerhafte Auslagerung, kein Server-Absturz).
