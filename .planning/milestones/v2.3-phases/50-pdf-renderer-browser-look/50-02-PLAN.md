**Layout-Algorithmus** (D-50-01 Hybrid Stack + Startkonstanten aus RESEARCH.md §Architecture Patterns):
- Landscape A4 (297×210mm)
- Header oben (Titel links Bold, Timestamp rechts 9pt) — 1 Zeile
- Tages-Header darunter (Mo/Di/Mi/…/So bzw. Mo–Sa je nach D-50-08-Sichtbarkeit)
- Grid darunter: `n_days` Spalten, dynamische Breite `(PAGE_WIDTH_MM - MARGIN_LEFT - MARGIN_RIGHT) / n_days`
- Pro Spalte: Slots als Boxen mit `add_rect(Rect::with_mode(PaintMode::Stroke))` gerendert, `height_mm = SLOT_BASE_MM + duration_hours * SLOT_STEP_MM`
- Slots sortiert (D-50-02: from, to, id), vertikal gestapelt mit `SLOT_GAP_MM` Abstand
- Namen innerhalb Box alphabetisch (D-50-06), Freiwillige mit ` (freiwillig)` (D-50-07)
- Overflow via `+ N weitere` (D-50-03, D-50-04)
- Slot-Rahmen 0.4pt schwarz, kein Fill (D-50-10)

**Metadata bleibt** (D-50-13): `FIXED_METADATA_TIMESTAMP` und `PDF_PRODUCER` als Konstanten für die deterministischen Trailer-Bytes.

**Umlaut-Kompatibilität** (RESEARCH §Common Pitfalls Pitfall 3): printpdf 0.7 nutzt WinAnsiEncoding via lopdf, deutsche Umlaute werden korrekt gerendert. Für Test-Assertions bleibt `encode_ascii_to_pdf_hex` das Standardwerkzeug für ASCII-Strings (Header, Timestamp, Zeitlabels, Test-Sales-Person-Namen ohne Umlaute).

Purpose: Der PDF-Look entspricht dem Browser-Wochenraster (PDF-01), der Timestamp erscheint sichtbar (PDF-02), byte-Determinismus ist aufgehoben (D-50-13), alle Backend-Gates bleiben grün.

    ```
        slot: &ShiftplanSlot,

        // 3) Namen darunter, alphabetisch, Overflow via `+ N weitere` (D-50-04).
        let mut name_y = label_y - LINE_HEIGHT_MM;
        let name_bottom_limit = box_y_bottom + SLOT_PADDING_MM;
        for (i, name) in sorted_names.iter().enumerate() {
            let next_y = name_y - LINE_HEIGHT_MM;
            let is_last = i == sorted_names.len() - 1;
            if next_y < name_bottom_limit && !is_last {
                // Nächster Name würde nicht passen und wir sind nicht am Ende: Overflow-Suffix.
                let remaining = sorted_names.len() - i;
                let overflow_text = format!("+ {} weitere", remaining);
                layer.use_text(overflow_text, NAME_FONT_PT, Mm(box_x + 1.5), Mm(name_y), font);
                return;

            if name_y < name_bottom_limit {
                // Auch dieser Name passt nicht mehr — trotzdem kein Rendering weiter.
                return;
            }
            layer.use_text(name.clone(), NAME_FONT_PT, Mm(box_x + 1.5), Mm(name_y), font);

        }
    ```

    F) **Day-Column-Rendering mit Slot-Overflow** (D-50-03 Slot-Overflow, RESEARCH §Pattern 6):
    ```
    fn render_day_column(
        layer: &printpdf::PdfLayerReference,
        sorted_slots: &[&ShiftplanSlot],
        sales_persons: &[SalesPerson],
        col_x: f32,
        col_width_mm: f32,
        grid_top_y: f32,  // Y-Position (oberer Rand des Slot-Bereichs)
        avail_height: f32,
        font: &printpdf::IndirectFontRef,
        font_bold: &printpdf::IndirectFontRef,
    ) {
        let grid_bottom_y = grid_top_y - avail_height;
        let mut y_cursor = grid_top_y; // Beginn oben; jede Box startet mit oberer Kante hier.
        let mut rendered_count: usize = 0;

        for (i, slot) in sorted_slots.iter().enumerate() {
            let duration_hours = compute_slot_duration_hours(&slot.slot);
            let box_h = compute_slot_height_mm(duration_hours);
            let box_bottom = y_cursor - box_h;
            if box_bottom < grid_bottom_y {
                // Diese Box passt nicht mehr → Overflow-Marker in vorheriger Box (D-50-03).
                let remaining = sorted_slots.len() - i;
                // Wir haben rendered_count Boxen bereits gerendert; Suffix in der zuletzt gerenderten.
                // Vereinfachter Ansatz: `+ N weitere` an leere Position unter grid_top_y für die
                // Spalte schreiben, wenn KEINE Box gerendert wurde; sonst in Zone unter der
                // letzten Box (analog Browser overflow-hidden).
                if rendered_count == 0 {
                    // Keine Box passt in den verfügbaren Platz — dennoch Marker in Grid-Kopf.
                    layer.use_text(
                        format!("+ {} weitere", remaining),
                        NAME_FONT_PT,
                        Mm(col_x + 1.5),
                        Mm(grid_top_y - LINE_HEIGHT_MM),
                        font,
                    );
                } else {
                    // Marker unter dem unteren Rand der letzten Box (etwas Padding).
                    layer.use_text(
                        format!("+ {} weitere", remaining),
                        NAME_FONT_PT,
                        Mm(col_x + 1.5),
                        Mm(y_cursor - LINE_HEIGHT_MM * 0.5),
                        font,
                    );
                }
                return;
            }
            let names = build_slot_name_list(slot, sales_persons);
            render_slot_box(layer, slot, &names, col_x, box_bottom, col_width_mm, box_h, font, font_bold);
            y_cursor = box_bottom - SLOT_GAP_MM;
            rendered_count += 1;
        }
    }
    ```
