<objective>
Finalisiert die Aufrufer-Verantwortung nach D-50-12: Ersetzt die Übergangs-Bridge `OffsetDateTime::now_utc()` aus Wave 2 (50-02 Task 4) durch die korrekte `now_local()`-mit-UTC-Fallback-Logik, extrahiert diese in eine testbare `pub(crate) fn resolve_render_timestamp()`, und fügt den letzten D-50-16-Test `now_local_fallback_to_utc_on_indeterminate_offset` in `service_impl/src/test/pdf_shiftplan.rs` an.

**Warum eine dedizierte Fn:** Der `IndeterminateOffset`-Fehler von `time::OffsetDateTime::now_local()` ist auf Linux/NixOS mit Tokio-Multi-Thread durch `localtime_r` (thread-safe) nicht simulierbar ohne `unsafe { set_local_offset }` — der Test verifiziert stattdessen, dass `resolve_render_timestamp()` NIE paniced und immer einen plausiblen `OffsetDateTime` liefert (Smoke-Test für die `unwrap_or_else`-Verkabelung). Das reicht als Nyquist-Signal: würde jemand versehentlich `.unwrap()` einführen, wäre der Test rot in JEDEM Deployment ohne funktionierendes Local-TZ (z.B. Docker ohne TZ-Env, minimal-Alpine, CI ohne TZ-Data).

**Sanity-Check auf `pdf_export_scheduler.rs`:** Muss weiterhin `PdfShiftplanService` konsumieren (D-49-08 aus Phase 49) — kein direkter `pdf_render`-Aufruf. Per grep verifizieren, kein Code-Change nötig.

Purpose: Aufrufer-Verantwortung nach D-50-12 vollständig umgesetzt; single injection point in `PdfShiftplanService`; Backend-Tests und Clippy-Gate grün; Phase 50 formal abgeschlossen (nur noch UAT via D-50-17-Button-Klick als Post-Gate).
