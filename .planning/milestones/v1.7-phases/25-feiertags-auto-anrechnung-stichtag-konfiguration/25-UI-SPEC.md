## Spacing Scale

All values from the existing project Tailwind config (4px base scale). No new tokens
are introduced — the new card reuses the same spacing as the Phase 24 toggle card.

| Token | Tailwind class | px value | Usage in settings page |
|-------|---------------|----------|------------------------|
| xs | `gap-1` | 4px | Label-description vertical gap within a row |
| sm | `gap-2` | 8px | Input→button gap |
| md | `p-4`, `px-4`, `py-4`, `gap-3` | 16px / 12px | Card padding; action-row gap |
| lg | `md:px-6` | 24px | Responsive page side padding (tablet) |
| xl | `lg:px-8`, `pb-4` | 32px / 16px | Responsive page side padding (desktop); h1 bottom |
| 2xl | `max-w-5xl mx-auto` | layout constraint | Page container max-width (inherited, not a gap) |

Exceptions: Date input field width is constrained to `max-w-[200px]` on its wrapper
div (not an 8-pt multiple) so that the native date picker does not stretch full-width
on desktop. This matches browser UX conventions for fixed-format date fields.

## Typography

The full typography scale is declared in `tailwind.config.js:56-63`. This phase uses
a strict subset matching the Phase 24 settings page.

| Role | Tailwind token | px / weight / line-height | Usage |
|------|---------------|--------------------------|-------|
| Page heading | `text-h2 font-semibold` | 18px / 600 / 24px | `<h1>` "Settings" (existing, unchanged) |
| Field label (semantic) | `text-body text-ink font-semibold` | 14px / 600 / 20px | Section feature-name label (same as toggle label in Phase 24) |
| Field description | `text-small text-ink-soft` | 12px / 500 / 16px | One-liner explanation below the feature label |
| Input text | `text-body` (via `SHARED_INPUT_CLASSES`) | 14px / 400 / 20px | Date input field content |
| Micro label (Field atom) | `text-micro text-ink-soft uppercase` | 11px / 600 / 14px | `Field` component uppercase label above the input (NOT used here — see Layout section) |
| Feedback / hint text | `text-small text-ink-muted` | 12px / 500 / 16px | "Saved." / "Not set — automation is off." |
| Error feedback | `text-bad text-small` | 12px / 500 / 16px | Save error message |

Weights used: 400 (body), 500 (small), 600 (semibold label / heading / micro).
**No new weights introduced** — every weight is baked into the existing
`tailwind.config.js:56-63` fontSize tokens (`small` carries `fontWeight: '500'`,
`body` `'400'`, `micro`/`lg`/`h2`/`h1` `'600'`). This phase adds zero typography
tokens and zero `font-*` utility overrides.

> **Accepted deviation (UI-checker Dimension 4, "max 2 weights"):** The shifty-dioxus
> design system pre-dates this phase and legitimately uses three weights via its
> baked-in fontSize tokens. Collapsing to two would force this control to override a
> token weight (e.g. `text-small font-normal`), diverging from how `text-small`
> renders app-wide and breaking the Phase-24 settings-page consistency mandated by
> D-25-06. The 3-weight count is the project baseline, not a net-new decision —
> deviation accepted, no override applied.

```
        value: date_str,   // ImStr — ISO YYYY-MM-DD if set, else ""
        on_change: update_date_signal,
      }

    // Row C: Action row (Save + Clear + inline feedback)
    div { class: "flex flex-row items-center gap-3" }
      button {
        class: "px-3 py-2 rounded-md border border-border text-ink text-body bg-surface hover:bg-surface-alt",
        disabled: is_saving,
        onclick: on_save,
        "{i18n.t(Key::SettingsHolidayAutoCreditSave)}"
      }
      button {
        class: "px-3 py-2 rounded-md border border-border text-ink-soft text-body bg-surface hover:bg-surface-alt",
        disabled: is_saving || date_str.is_empty(),
        onclick: on_clear,
        "{i18n.t(Key::SettingsHolidayAutoCreditClear)}"
      }
      // Inline feedback — reuses existing keys
      match *save_result.read() {
        Some(true)  => span { class: "text-small text-ink-muted" } // SettingsSaved
        Some(false) => span { class: "text-bad text-small" }       // SettingsSaveError
        None        => empty
      }

    // Row D: Unset hint (shown ONLY when date_str is empty after load)
    if date_str_loaded_empty {
      span { class: "text-small text-ink-muted" }
        "{i18n.t(Key::SettingsHolidayAutoCreditUnsetHint)}"
    }
```

`mt-4` (16px top margin) separates Card 2 from Card 1 without adding a section header.

## State Machine

| State | Signal values | Visible UI |
|-------|--------------|------------|
| Loading | `toggle_resource` pending | Date input shows `value=""` (browser empty state); both buttons present but effectively no-op until load completes |
| Unset (automation off) | `date_str = ""`, `date_str_loaded_empty = true` | Empty date input + unset hint text below action row |
| Set | `date_str = "YYYY-MM-DD"`, `date_str_loaded_empty = false` | Date input shows ISO value (browser renders locale-formatted); no unset hint; Clear button enabled |
| User editing | `date_str` updated by `oninput` signal | Save button enabled; Clear still visible if was previously set |
| Saving | `saving = true` | Both buttons `disabled`; no spinner (matching Phase 24 pattern) |
| Saved | `save_result = Some(true)` | Inline "Saved." in `text-small text-ink-muted` to the right of buttons; auto-clears after next interaction |
| Save error | `save_result = Some(false)` | Inline error in `text-bad text-small`; date input value unchanged |

**Note on programmatic date input (MEMORY `reference_dioxus_browser_test_date_inputs.md`):**
Setting `value` on `<input type=date>` programmatically does NOT reliably trigger
Dioxus signal updates in WASM. The Save button MUST be enabled whenever the
date input contains any non-empty string — do NOT gate on "value changed from loaded
state." Validation occurs on Save (ISO date parse check before PUT).

## Interaction Contract

### Save flow ("Save date" button)

1. User selects or types a date in the `<input type="date">` field.
2. `oninput` updates a local `date_str` signal (ImStr, ISO format `YYYY-MM-DD`).
3. User clicks "Save date".
4. `on_save` handler:
   a. Parse `date_str` as ISO date — if parse fails, set `save_result = Some(false)` and return early.
   b. Set `saving.set(true)`, `save_result.set(None)`.
   c. `spawn` async: call backend (PUT `/toggle/holiday_auto_credit/value` or equivalent new endpoint to persist the ISO date string in the `value` column).
   d. On `Ok(())`: set `save_result = Some(true)`, `saving.set(false)`, `date_str_loaded_empty.set(false)`.
   e. On `Err(_)`: set `save_result = Some(false)`, `saving.set(false)`.

### Clear flow ("Clear (disable)" button)

1. User clicks "Clear (disable)".
2. `on_clear` handler:
   a. Set `saving.set(true)`, `save_result.set(None)`.
   b. `spawn` async: PUT null/empty value to disable the stichtag.
   c. On `Ok(())`: set `date_str.set(ImStr::from(""))`, `date_str_loaded_empty.set(true)`, `save_result = Some(true)`, `saving.set(false)`.
   d. On `Err(_)`: set `save_result = Some(false)`, `saving.set(false)`.
3. No confirmation dialog — cleared inline; effect is reversible (user can save a new date).

### Load flow (page mount)

1. `use_resource` fetches current toggle value via existing or new GET endpoint (returns `Option<String>` — None if unset).
2. `use_effect` on resource resolution:
   a. If `Some(date_str)` → set `date_str` signal to ISO date, `date_str_loaded_empty = false`.
   b. If `None` → set `date_str = ""`, `date_str_loaded_empty = true`.
3. Signals: mirror pattern from Phase 24 `toggle_resource` → `use_effect` sync.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Date input | No explicit `aria-label` needed — input is associated to the feature label via proximity; browser `<input type=date>` has native label via containing `<label>` or visible label span. |
| Save button | No `aria-pressed` (not a toggle). `disabled` attribute when `saving = true`. |
| Clear button | `disabled` when `date_str` is empty (nothing to clear) or when `saving = true`. |
| Feedback span | Inline, immediately adjacent to the action buttons — screen reader discovers it in DOM order without `aria-live`. |
| Admin gate | Existing `if !is_admin { return not-authorized rsx! }` guard is unchanged. |
