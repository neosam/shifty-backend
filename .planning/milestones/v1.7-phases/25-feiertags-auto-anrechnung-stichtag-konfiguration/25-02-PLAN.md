<objective>
Implement the holiday auto-credit as derive-on-read in the reporting service: derive each
employee's holiday hours from `special_day` (type Holiday) plus the contract valid on that day,
gate them behind the configurable cutoff (read from the `holiday_auto_credit` toggle value
created in 25-01), skip any holiday already covered by a manual ExtraHours(Holiday), and inject
the result into the exact same `holiday_hours`/`absense_hours` aggregation that manual entries
use — so the effect is byte-for-byte identical to a manual ExtraHours(Holiday). Bump the billing
snapshot schema version because this changes the computation of `BillingPeriodValueType::Holiday`.

Purpose: HOL-01/02/03, HCFG-01 (gate), HCFG-03 (conflict), HSNAP-01. Implements D-25-01
(derive-on-read), D-25-02 (holiday_hours helper + has_day_of_week), D-25-03 (manual wins),
D-25-07 (bump 10->11), D-25-08 (year-view untouched), D-25-09 (cutoff gates only HOL).

Output: reporting derive-on-read across all three injection points, the main.rs DI reorder,
and the snapshot version bump + locking-test update. Behavioral proof lands in plan 25-04.
</objective>
