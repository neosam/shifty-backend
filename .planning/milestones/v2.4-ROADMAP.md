## Overview

Fokus-Milestone auf einer einzelnen Semantik-Ergänzung. An Kurzen Tagen
(`special_day.ShortDay` mit Cutoff-Uhrzeit) werden Slots, die den Cutoff
überlappen, dynamisch auf `[slot.start, cutoff]` gekürzt — sowohl im
Schichtplan-Rendering (WeekView, PDF) als auch in der Ist-Stunden-
Berechnung (Reporting, Booking-Information, Balance). Slots komplett
hinter dem Cutoff verschwinden. Soll-Stunden bleiben unverändert.

Die Kürzung wirkt view-layer / dynamisch — keine DB-Änderung an Slots
oder Bookings, kein Snapshot-Bump. Ein admin-konfigurierbarer Stichtag
(`shortday_slot_clipping_active_from`, ISO-Datum via `ToggleService`)
schützt historische Balance-Views: ohne gesetzten Wert bleibt die
Kürzung deaktiviert (Rollout-Default, Legacy-Verhalten); mit gesetztem
Wert wirkt sie ausschließlich für Bookings mit
`booking_date >= active_from`.

Vier Backend-Aggregat-Ketten wurden erweitert (Chain A' BlockService,
Chain B ShiftplanWeek/PDF, Chain C BookingInformation, Chain D
ShiftplanReport). Chain D wurde dabei Rust-Layer-refaktoriert (raw-row
DAO statt SUM-SQL) — das räumte einen pre-existing `/60.0`-SQL-Bug
in den alten SUM-Queries mit ab. Die kanonische Clip-Funktion
`Slot::clip_to` sitzt auf `service::slot::Slot` und ist pure +
unit-getestet. DTO `ShiftplanSlotTO.effective_to` (Wrapper-Field) trägt
den geclippten Wert ans Frontend + den PDF-Renderer. Fat Backend, Thin
Client (D-51-02): das Frontend enthält null Clip-Logik, es rendert
`effective_to`.

## Milestone Summary

**Key accomplishments:**

1. **Kanonische `Slot::clip_to` als pure Value-Methode** — auf
   `service::slot::Slot` (D-51-01), vier D-04-Grenzfall-Tests
   (`slot.end < cutoff` / `slot.end == cutoff` / `slot.start < cutoff <
   slot.end` / `slot.start >= cutoff`), keine Panics, keine unwrap.

2. **Toggle-Stichtag + `shortday_gate`-Helper** (SHC-06 BE) — additive
   Migration `20260704000001_seed-shortday-slot-clipping-toggle.sql`
   (`INSERT OR IGNORE`, `enabled=0`, `value=NULL` → Rollout-Default
   „Kürzung aus"). `shortday_gate::parse_active_from` + `should_clip` +
   `resolve_active_from_for_week` als pure Rust-Helper (kein async, kein
   DAO); Präzedenz HCFG-02.

3. **Vier BE-Aggregat-Ketten clippen konsistent** — Chain A' BlockService
   (iCal + insufficient-Report via einem shared `clip_slot_for_week` +

   `shiftplan.rs`), Chain C `booking_information` (ersetzt denselben
   Filter-statt-Clip-Bug), Chain D `ShiftplanReport` als
   Rust-Layer-Refactor (raw-row DAO + Aggregation im Service, entfernt
   die alten SUM-Queries mit dem pre-existing `/60.0`-Bug).

4. **DTO-Wrapper-Field `ShiftplanSlotTO.effective_to`** (D-51-09) — trägt
