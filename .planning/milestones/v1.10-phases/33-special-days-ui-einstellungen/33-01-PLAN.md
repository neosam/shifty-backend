Add the Range/Jahr read endpoint to the Special-Days backend (D-33-05) and lock in test
coverage for the existing-but-untested create/delete permission behavior. The endpoint is a
direct end-to-end clone of the existing `for-week` chain (DAO → Service → REST → ApiDoc).

create/delete (D-33-01 / SPD-04) plus the nil-id guard (SPD-01) become regression-protected.

delivers SPD-02's for-year read endpoint and asserts the SPD-04 / D-33-01 shiftplanner
permission gate, and additionally regression-tests the SPD-01 nil-id guard and SPD-03
not-found guard at the service tier. The i18n (SPD-04) and UI portions of SPD-01/02/03 land
in Plans 02-04. Hence all four IDs appear in this plan's `requirements`.
