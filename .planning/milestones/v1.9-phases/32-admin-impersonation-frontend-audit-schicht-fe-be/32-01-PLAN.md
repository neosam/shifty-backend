<objective>
Add the backend audit layer for admin impersonation: a `RealUser` extension carrying the
real admin identity, a single central tower middleware that logs every mutating request
made under impersonation, explicit start/stop audit lines, and documentation of the
already-implemented admin-gate two-path invariant plus the P10 known limitation. The
impersonation mechanism (endpoints, session.impersonate_user_id, resolve_session_user_id,
admin-gate against raw session.user_id) ALREADY EXISTS — this plan adds only the audit
schicht and its documentation/tests. No new endpoints, no Authentication<Context>
signature change, no DB/migration, no snapshot bump.

Purpose: Satisfy IMP-03 (writes under impersonation auditable with the real admin
identity) and the IMP-01 gate guarantee (non-admin cannot impersonate), with regression
tests pinning SC3 and SC5.
Output: RealUser newtype + pure helpers, the audit middleware, start/stop tracing,
the two-path/P10 doc, and an integration-test module.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /admin/impersonate | Untrusted caller requests to act as another user; must be gated against the REAL caller identity. |
| session → request extensions | context_extractor derives the effective Context and the RealUser audit identity from a verified server-side session, never from client-supplied values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-32-01 | Elevation of Privilege | start_impersonate handler | critical | mitigate | Admin check uses raw session.user_id (D-32-02); Test 1 asserts non-admin → 403, admin → 200. |
| T-32-02 | Repudiation | mutating requests under impersonation | high | mitigate | Central audit_impersonated_writes middleware logs real_user + acting_as for all POST/PUT/PATCH/DELETE; explicit start/stop tracing (D-32-01). |
| T-32-03 | Spoofing | RealUser identity | medium | mitigate | RealUser is sourced only from the verified session.user_id inside context_extractor; never read from client input. |
| T-32-04 | Elevation of Privilege | admin-only endpoints during impersonation of a non-admin | low | accept | By-design lockout (P10, D-32-02a); documented; stop remains available via raw session.user_id. |
| T-32-SC | Tampering | npm/pip/cargo installs | n/a | accept | No new dependencies in this phase (RESEARCH SUMMARY: zero new deps); no package-install tasks. |
</threat_model>
