<objective>
Make days with the selected person's own absence periods appear proactively as the existing
"Nicht Verfügbar" (discourage) cell in the shiftplan grid — before a booking is attempted —
by mirroring the existing unavailable-day marker for absence date ranges (SHP-01).

This is a frontend-only join over the existing `/absence-period` API: no backend, no new
endpoint/DTO, no new UI, no new marker type. The absence-derived weekdays are union-merged
into the same `discourage_weekdays: Rc<[Weekday]>` that already drives the red cell.

Purpose: close the verified gap "Eigener Urlaub markiert nicht als Nicht-Verfügbar im
Schichtplan" — `discourage_weekdays` today only feeds from `sales_person_unavailable`, so
absences never mark the grid. The marker must behave EXACTLY like a stored unavailable day
(same source person, same discourage rendering, same merge target), with zero category drift
versus the post-booking `BookingOnAbsenceDay` warning.

Output: a new pure, unit-tested helper module (`service/absence_marker.rs`), a new
`person_absences` signal, a new guarded `reload_absence_days` closure, and the union-merge
into `discourage_weekdays`.
</objective>
