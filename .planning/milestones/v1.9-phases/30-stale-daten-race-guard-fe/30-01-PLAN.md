<objective>
Eliminate the stale-week race in the shiftplan summary area: when the user clicks
rapidly through weeks, the three async week loaders must never let a late-arriving
result for an old week overwrite (or be rendered for) the currently selected week.

Approach (locked by 30-CONTEXT, D-30-01..04): a single global "currently selected
(year, week)" truth (SELECTED_WEEK), set synchronously on every week change before
loads are dispatched. Each of the THREE loaders compares the (year, week) it loaded
for against SELECTED_WEEK AFTER its await and writes its store only on a match,
otherwise silently drops the result (no error, no log). A render-guard additionally
hides summary cards whose store data does not belong to the selected week. The
match check is a pure, cargo-test-covered predicate.

Purpose: SHP-02 — the weekly summary cards must always reflect only the selected week,
with no transient cross-week flicker, fixed atomically across all three loaders (no
partial fix).
Output: one new shared guard module + predicate (with tests), three guarded loaders,
one render-guard, mod registration. Frontend-only; no backend, no new endpoint, no
new dependency.
</objective>
