## Overview

Die Paid-Capacity-Grenze (`max_paid_employees` pro Slot/Woche) wurde von einem rein
visuellen Soft-Hinweis (v1.1/Phase 5, Phase 23) zu einem **global konfigurierbar
durchsetzbaren Limit**: admin-schaltbar hart/weich, im harten Modus rollenbasiert
(nur die Shiftplanner-Rolle darf überziehen), mit deutlicherer Overage-Anzeige im
Wochenplan. Default = weich (keine Regression). Mitgefixt wurde das Buchungs-
Permission-Gate (Shiftplanner ∨ self statt HR ∨ self).

## Milestone Summary

**Key Decisions:**

- Globaler Modus über bestehenden `ToggleService` statt `feature_flag` (D-24-01) —
  Rationale: `feature_flag` ist für Rollout-Gating, nicht für eine bleibende
  Admin-Konfiguration; Toggle-Tabelle ist die passende Persistenz.
- Pre-Persist-Hard-Block im Business-Logic-Tier (`ShiftplanEditService`), Toggle frisch
  pro Buchung gelesen (D-24-08) — Rationale: Enforcement gehört vor `booking_service.create`,
  und die Service-Tier-Konvention verbietet dem Basic-Tier-`BookingService` Cross-Service-Deps.
- HTTP **409** statt 403 für Limit-Überschreitung (D-24-05) — Rationale: 409 Conflict
  unterscheidet die Kapazitäts-Kollision sauber von einer echten Permission-Verweigerung (403)
  und erlaubt dem Frontend ein gezieltes Inline-Handling.
- Strikt-größer-Grenzregel (`prospective > max` blockt) — Rationale: das Limit selbst
  bleibt erreichbar; nur das Überschreiten wird verhindert.
- `current_paid_count` für die Overage-Sektion aus un-gegatetem
  `get_all_paid(Authentication::Full)` zählen, per-Booking `is_paid` aber weiterhin
  gegated — Rationale: D-24-03 „alle Rollen" braucht den Count, ohne *wer* bezahlt ist zu leaken.

**Issues Resolved (während UAT gefunden + gefixt):**

- **Bug A** — `/toggle`-Pfadfamilie fehlte in der Dev-Proxy-Allowlist
  (`shifty-dioxus/Dioxus.toml`): Settings-Toggle war im Dev funktionslos
  (GET → SPA-`index.html`, PUT → 405). Fix: `[[web.proxy]] backend =
  ".../toggle"` ergänzt.
- **Bug B** — Overage-Sektion war nur für HR sichtbar (D-24-03 verletzt):
  `current_paid_count` wurde aus dem HR-gegateten `is_paid` berechnet → für
  Nicht-HR immer 0. Fix: Count aus `get_all_paid(Authentication::Full)` +
  Regressionstest `test_current_paid_count_correct_for_non_hr_caller`.
- **D-24-04 Permission-Gate** — Buchungs-Gate korrigiert von `HR ∨ self` auf
  `Shiftplanner ∨ self`.
