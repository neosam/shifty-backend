<objective>
Build the data layer for the per-(person, year) signed vacation-entitlement offset: a new soft-delete table, its DAO trait + sqlite implementation, and a new HR-gated **Basic** service (`VacationEntitlementOffsetService`) that performs CRUD on it. This is the snapshot-SAFE half of Phase 28 — it touches no reporting/billing computation.

Purpose: Establish a clean, isolated aggregate (D-28-01) and Basic service (D-28-06) that the Business-Logic `VacationBalanceService` (Plan 28-02) will consume to add the offset after rounding. Keeping the offset service Basic (no domain-service deps) guarantees no DI cycle.

Output: A migrated table, DAO trait + sqlite impl, service trait + impl, and a CRUD + HR-gate test module — all compiling and green under `cargo build`/`cargo test`/`cargo clippy --workspace -- -D warnings`.
</objective>
