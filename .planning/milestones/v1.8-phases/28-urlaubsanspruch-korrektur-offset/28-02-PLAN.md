<objective>
Wire the signed offset into the read path and expose HR-gated write endpoints. The Business-Logic `VacationBalanceService` reads the Basic offset service (from Plan 28-01), adds the offset after `.round()`, and conditionally exposes the offset breakdown only to HR callers (API-level hiding). A small dedicated HR-gated REST CRUD endpoint lets HR set/clear the offset.

Purpose: Deliver D-28-02 (offset-after-round → flows into remaining), D-28-03 (server-side field hiding + effective-always), and D-28-06b (HR-gated REST). This makes the effective entitlement correct for both roles while never leaking the raw offset to self-only callers.

Output: Updated domain + TO + service computation + DI wiring + a REST CRUD endpoint, all compiling, with backend tests for offset-calc, delta-behavior, and API-hiding.
</objective>
