# v2.3 Roadmap Archive — PDF-Export: Browser-Look & Download-Button

**Shipped:** 2026-07-04
**Phases:** 49–50 (2 phases, 8 plans)
**Requirements:** PDF-01, PDF-02, PDF-03, PDF-04, PDF-05
**Character:** Kleiner Fix-Milestone auf dem v2.2-PDF-Export. Der v2.2-Renderer
produzierte praktisch unlesbare PDFs; v2.3 machte daraus eine browser-ähnliche
Wochenansicht und legte einen On-Demand-Download-Button auf die Schichtplan-
Seite. Kein Snapshot-Bump (bleibt 12), keine Migration, keine neue Cargo-Dep.
WebDAV-Scheduler aus Phase 48 nutzt den neuen Renderer nach Phase 50 automatisch.

**Reihenfolge-Rationale:** Erst der Button (Phase 49) mit dem alten,
unlesbaren Renderer, dann der Renderer-Rewrite (Phase 50). So konnte das
Rendering-Ergebnis in Phase 50 direkt per Button-Klick verifiziert werden.

## Phase 49: On-Demand-Download-Button (BE + FE)

**Goal:** Auf der Schichtplan-Seite gibt es einen PDF-Download-Button, der die
aktuell im UI selektierte Kalenderwoche des ausgewählten Shiftplans für jeden
authentifizierten User ausliefert — aber nur sichtbar, wenn
`week_status ∈ {Planned, Locked}`.

**Depends on:** Nichts Neues (nutzt den v2.2-Renderer 1:1).
**Requirements:** PDF-03, PDF-04, PDF-05
**Plans:** 5/5 plans complete
**Completed:** 2026-07-03

### Success Criteria

1. Neuer REST-Endpoint `GET /shiftplan/{shiftplan_id}/{year}/{week}/pdf`
   liefert das PDF mit Dateiname `schichtplan-{JJJJ}-KW{NN}.pdf`
   (Content-Disposition-Header); auth-required, aber KEIN Admin-Gate
   (Employee-Auth liefert 200).
2. Backend gibt HTTP 409 zurück, wenn `week_status ∈ {Unset, InPlanning}`
   (Defense-in-Depth für Race-Case).
3. Frontend-Button in `shifty-dioxus/src/page/shiftplan.rs` sitzt neben dem
   iCal-Button und wird nur sichtbar gerendert, wenn
   `week_status ∈ {Planned, Locked}` (kein disabled-Zustand, kein Tooltip,
   kein Fehler-Toast). Button lädt die aktuell im UI selektierte KW des
   ausgewählten Shiftplans. i18n-Label in de/en/cs.
4. DRY: neuer Business-Logic-Service `PdfShiftplanService` kapselt
   `ShiftplanViewService` + `SalesPersonService` + `WeekStatusService` +
   `pdf_render`. `PdfExportScheduler` wurde refactored, damit er diesen
   Service konsumiert statt inline zu orchestrieren (einheitlicher
   Assemble-Pfad für On-Demand + Cron).

## Milestone Summary

### Delivered

- Neuer REST-Endpoint `GET /shiftplan/{id}/{year}/{week}/pdf` mit
  auth-Gate (kein Admin-Gate) und `WeekStatus`-Defense-in-Depth-409.
- `PdfShiftplanService` als Business-Logic-Tier-Assembler
  (ShiftplanView + SalesPerson + WeekStatus + pdf_render).
- `PdfExportScheduler` refactored — delegiert Assemble jetzt an
  `PdfShiftplanService` (single Assemble-Pfad On-Demand + Cron).
- Frontend-Download-Button neben iCal auf `page/shiftplan.rs`, visibility-
  gated auf `WeekStatus ∈ {Planned, Locked}`; pure Predicate-Fn mit 8-Fall-
  Testmatrix; i18n de/en/cs.
- Kompletter PDF-Renderer-Rewrite: Hybrid-Stack-Layout, sichtbare
  Slot-Rahmen (`add_rect` + `PaintMode::Stroke`), dynamische Sonntag-Spalte,
  Header-Timestamp „Erstellt am DD.MM.YYYY HH:MM Uhr", alphabetische Namen
  mit `(freiwillig)`-Suffix (später auf User-Wunsch entfernt), Overflow-
  Marker.
- `resolve_render_timestamp()` mit `time::OffsetDateTime::now_local()` +
  UTC-Fallback + `tracing::warn!`-Log (D-50-12).
- Cargo-Feature `local-offset` auf existierendem `time` — keine neue Crate.

### Key Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D-49-03 | Handler-level 409 pre-check via `WeekStatusService` + stable JSON error code `week-not-releasable` | Defense-in-Depth wenn FE-Status zwischen Signal-Update und Klick wechselt |
| D-49-08 | Scheduler delegiert Assemble an `PdfShiftplanService` | DRY: einheitlicher Pfad für On-Demand + Cron |
| D-49-13 | Pure Visibility-Predicate `should_show_pdf_button(status, shiftplan_id) -> bool` | Testbar ohne Dioxus VirtualDom (8-case matrix) |
| D-49-14 | `PdfDownload` i18n-Key mit Wert `"PDF"` in de/en/cs | Kein sprach-spezifischer Text — Icon-artige Beschriftung |
| D-49-15/16 | Doku-Deviation textuell verankert (REQUIREMENTS + ROADMAP) | Audit-Trail für D-49-04-Cutover |
| D-50-01/02 | Hybrid-Stack-Layout (base + duration_hours × step) statt fixem Grid | Slots visuell proportional zur Länge |
| D-50-08 | Dynamische Sonntag-Spalte analog `has_sunday` im WeekView | PDF konsistent zum Browser |
| D-50-10 | Slot-Rahmen via `add_rect(Rect::with_mode(PaintMode::Stroke))` + `save_graphics_state`/`set_outline_thickness`/`restore_graphics_state` | printpdf-0.7-idiomatisch, lokal begrenzte Linien-Dicke |
| D-50-11 | Renderer 5-Parameter-Signatur (`render_timestamp` als Argument) | Renderer bleibt pure Funktion — Aufrufer injizieren Uhrzeit |
| D-50-12 | `now_local()` mit UTC-Fallback + `warn!`-Log bei `IndeterminateOffset` | Graceful Degradation für TZ-Detection-Failure |
| D-50-17 | Human UAT: visueller PDF-Layout-Check via Phase-49-Button gegen reale Woche | Ausdruck-ohne-Digital-Referenz-Kriterium (PDF-01) |
| Design-Deviation (post-UAT) | `+ N weitere` Overflow-Marker (D-50-03/04) nachträglich entfernt; Boxen wachsen mit, Namen komma-separiert; `(freiwillig)`-Suffix (D-50-06/07) im Fix `a484f74` entfernt | User-Feedback im UAT: paid/unpaid irrelevant im PDF, Overflow im v2.3-Datenvolumen nicht nötig |
