## v1.8 Requirements

### Freiwilligen-Auswahl in den Abwesenheits-Selektoren (VOL-SEL)

- [x] **VOL-SEL-01**: Auf der Abwesenheitsseite lassen sich auch Freiwillige
  (`sales_person.is_paid == false`) auswählen — sowohl beim Anlegen einer Abwesenheit
  (AbsenceModal) als auch im HR-Personenfilter (AbsenceFilterBar). Angestellte und
  Freiwillige erscheinen im selben Dropdown sichtbar getrennt (gruppierter Selector via
  native `optgroup`, Angestellte zuerst), inaktive bleiben ausgeblendet, leere Gruppen
  werden nicht gerendert; beide Gruppen-Beschriftungen in de/en/cs. Reines Frontend
  (gemeinsamer Helfer für beide Call-Sites, kein Copy-Paste). Phase 27.

### Urlaubsanspruch-Korrektur via Offset (VAC-OFFSET)

- [x] **VAC-OFFSET-01**: HR kann den berechneten Jahres-Urlaubsanspruch einer Person um
  einen signed **Offset (Korrektur-Delta)** anheben/senken (`entitled_effective =
  round(berechnet) + offset`, wirkt auf `remaining_days` durch). Der Offset ist signed,
  pro Person+Jahr persistiert, überlebt Vertragsänderungen (Delta, kein Override) und ist
  HR-gated (`HR_PRIVILEGE`). In der HR-Detailansicht an der „Vertragsanspruch"-StatBox
  gekennzeichnet+inline editierbar („berechnet {n} + Offset [x]", on-blur/Enter); für
  normale User unsichtbar (API-level Hiding — nur der Effektivwert). Neue Tabelle
  `vacation_entitlement_offset` + HR-gated REST CRUD + FE-Inline-Editor. Begleitend:
  Off-by-one-Proration-Fix + Snapshot-Bump 11→12. Phase 28 (BE+FE).
