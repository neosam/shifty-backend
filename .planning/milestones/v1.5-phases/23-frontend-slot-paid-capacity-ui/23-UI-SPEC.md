## Color

The project uses semantic CSS-variable-backed color tokens defined in `input.css` and aliased in
`tailwind.config.js`. No hex values are written directly in component code.

| Role | Token | Light value | Dark value | Usage in this phase |
|------|-------|-------------|------------|---------------------|
| Dominant surface (60%) | `--bg` / `bg-bg` | `#fbfbfc` | `#0e1014` | Page and dialog backdrop |
| Secondary surface (30%) | `--surface` / `bg-surface` | `#ffffff` | `#16191f` | Dialog panel, form inputs (`bg-surface` in FORM_INPUT_CLASSES) |
| Tertiary surface | `--surface-alt` / `bg-surface-alt` | `#f4f5f7` | `#1c2027` | Slot-cell alternating, day-header |
| Text dominant | `--ink` / `text-ink` | `#0e1117` | `#eef0f4` | Input values, form labels |
| Text muted | `--ink-muted` / `text-ink-muted` | `#6b7382` | `#7a8290` | Field hints (`text-micro text-ink-muted`) |
| Text soft | `--ink-soft` / `text-ink-soft` | `#3a4150` | `#b8bdc7` | Field label uppercase (`text-micro text-ink-soft`) |
| Border | `--border` / `border-border` | `#e6e8ec` | `#2a2f39` | Input borders, cell dividers |
| Border strong | `--border-strong` / `border-border-strong` | `#d0d3da` | `#3a4151` | Form input border (`FORM_INPUT_CLASSES`), inline-banner left accent |
| Warn fill | `--warn-soft` / `bg-warn-soft` | `#fef0d6` | `#3a2a14` | Week-view understaffed cell background (`missing=true, paid_overage=false`) |
| Warn text | `--warn` / `text-warn` | `#a65a08` | `#f0b766` | Inline-banner left border when warning; existing `valid_to` warning in slot editor |
| Bad fill | `--bad-soft` / `bg-bad-soft` | `#fde4e1` | `#3a1c18` | **NEW: paid-overage cell background** (D-23-03) + existing `discourage` cell background |
| Bad text | `--bad` / `text-bad` | `#b8281a` | `#ef6a5b` | Error paragraph in slot editor; `border-bad` for inline-hint left border |
| Accent | `--accent` / `accent` | `#3a4cd1` | `#8b97ff` | Focus ring (`input.css`: `.form-input:focus` border + `--accent-soft` shadow) |

**60/30/10 split:**
- 60% dominant: `--bg` (page) + `--surface` (dialog, card) + `--surface-alt` (alternating cells)
- 30% secondary: `--border`, `--border-strong`, `--ink-soft`, `--ink-muted`
- 10% accent: accent used exclusively for focus rings on form inputs (`.form-input:focus`)

**Accent reserved for:** form input focus ring only. No buttons, links, or decorative elements use accent in this phase.

**Semantic colors (paid-overage):**
- `bg-bad-soft` is the single background color for the paid-overage cell state (D-23-03).
  It is also the existing `discourage` (unavailability) color. Both states share the same
  light-red background, which is clearly distinguishable from the orange `bg-warn-soft`
  (understaffing). This is intentional per D-23-03 — the color signals "something wrong",
  not the specific cause.
- `bg-bad` (full dark red) is NOT used in this phase: it is not safelisted and is visually
  too strong for a background. If future requirements need paid-overage to look stronger
  than unavailability, add `"bg-bad"` to `tailwind.config.js safelist` and revisit.

**Warn banner (inline hint D-23-02):**
The inline banner in the slot editor uses `border-l-[3px] border-warn bg-warn-soft` for
a non-blocking warning (orange left accent, warm fill). This matches the project's
established inline-banner pattern (see `absences.rs` banner variants). Text inside: `text-body text-ink`.

## Interaction Contract

**Primary visual anchor of the slot-editor dialog is the new `max_paid_employees` Field input area** (the element this phase adds); the inline warn banner (when visible) is the secondary attention target, drawing the eye only when the entered limit is below the current paid count.

### FUI-02 — Max Paid Employees Field (Slot Editor)

**Location:** Inside `SlotEditInner`, inside the `Dialog`, after the `min_resources` Field and before the `has_errors` error paragraph.

**Input element spec:**
- Type: `<input type="number" min="0">`
- CSS classes: same as existing inputs — `FORM_INPUT_CLASSES` = `h-[34px] px-[10px] border border-border-strong rounded-md bg-surface text-ink text-body w-full min-w-0 form-input`
- Value display: `None` renders empty string `""`, `Some(n)` renders `n.to_string()`
- On input: empty string → `None`; parseable u8 → `Some(u8)`; parse failure → no update (silently ignore, consistent with `min_resources` pattern)
- Wrapped in: `Field { label: ..., hint: Some(...) }` — hint renders as `text-micro text-ink-muted` below the input

**Field label (i18n):**
- Key: `MaxPaidEmployeesLabel`
- De: "Max. bezahlte Mitarbeiter"
- En: "Max paid employees"
- Cs: "Max. placených zaměstnanců"

**Field hint (i18n, always visible below input):**
- Key: `MaxPaidEmployeesHint`
- De: "Leer = kein Limit"
- En: "Empty = no limit"
- Cs: "Prázdné = bez limitu"
- Rendered class: `text-micro text-ink-muted` (by `Field` component automatically)

**Inline warn banner (D-23-02, only when `current_paid_count > max_paid_employees`):**
- Rendered: a `div` placed immediately after the `max_paid_employees` Field block
- CSS: `border-l-[3px] border-warn bg-warn-soft rounded-md p-2.5 text-body text-ink`
- Content: i18n key `MaxPaidEmployeesOverageHint` with placeholders `{current}` and `{limit}`
  - De: "Aktuell {current} bezahlt (Limit: {limit})"
  - En: "Currently {current} paid ({limit} allowed)"
  - Cs: "Aktuálně {current} placených (limit: {limit})"
- Interpolation method: `i18n.t_m_rc(Key::MaxPaidEmployeesOverageHint, [("current", ...), ("limit", ...)].into())`
- Visibility condition: `props.slot.max_paid_employees.map_or(false, |n| props.current_paid_count > n)`
- Non-blocking: Save button remains enabled when banner is visible (D-23-02)
- No dialog, no disable — inline banner only (per project-wide preference, MEMORY.md)

### FUI-01 — Paid-Overage Cell Coloring (Week View)

**Location:** `WeekCellSlot` in `week_view.rs` — the `bg_class` variable at `:1037`.

**Logic (extended `cell_background_class`):**

```
paid_overage  = slot.max_paid_employees.map_or(false, |n| slot.current_paid_count > n)

cell_background_class(missing, discourage, paid_overage) ->
  if discourage    → "bg-bad-soft"   // unavailability: highest priority (unchanged)
  else if paid_overage → "bg-bad-soft"   // paid limit exceeded: second priority (NEW)
  else if missing  → "bg-warn-soft"  // understaffed: existing orange
  else             → ""              // normal
```

**Priority table (D-23-04):**

| discourage | paid_overage | missing | Result class | Visual |
|-----------|-------------|---------|--------------|--------|
| true | any | any | `bg-bad-soft` | Light red |
| false | true | true | `bg-bad-soft` | Light red (overage wins over understaffing) |
| false | true | false | `bg-bad-soft` | Light red |
| false | false | true | `bg-warn-soft` | Light orange |
| false | false | false | `""` | Default surface |

**Role visibility:** `paid_overage` computed unconditionally — not gated on `props.is_shiftplanner` (D-23-05). All user roles see the coloring.

**Existing badge unchanged:** The `filled/need` counter badge (`span` at `week_view.rs:1069-1073`) is not modified. No new badge or number added to the cell (D-23-03 deferred: "numerische Paid-Count-Anzeige nicht gewünscht").
