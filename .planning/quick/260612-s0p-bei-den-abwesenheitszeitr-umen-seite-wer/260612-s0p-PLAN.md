# Quick Task 260612-s0p: Abwesenheitszeiträume — nur bezahlte & aktive Sales Persons

## Beschreibung

Auf der Abwesenheitszeiträume-Seite (`shifty-dioxus/src/page/absences.rs`) werden in
den Personen-Dropdowns aktuell **alle** Sales Persons aufgelistet. Es sollen nur die
angezeigt werden, die **bezahlt** (`is_paid == true`) **und aktiv** (`inactive == false`)
sind.

Betroffen sind zwei Dropdowns:
1. `AbsenceModal` — Mitarbeiter-Auswahl beim Anlegen/Bearbeiten (Zeile ~1032)
2. `AbsenceFilterBar` — HR-Personenfilter (Zeile ~1239)

Hinweis: Die Filterung erfolgt **nur in den Dropdowns** (Auswahlpunkt), nicht an der
Datenquelle. Bestehende Absences von inzwischen inaktiven/unbezahlten Personen müssen
weiterhin korrekt mit Namen aufgelöst werden — daher bleibt die geladene Liste vollständig.
Dies entspricht dem etablierten Muster (`shiftplan.rs` filtert ebenfalls inline am Dropdown).

## Tasks

### Task 1: Testbare Filter-Helper + Anwendung in beiden Dropdowns
- **files:** `shifty-dioxus/src/page/absences.rs`
- **action:**
  - Pure-Function-Helper `is_selectable_employee(sp: &SalesPerson) -> bool`
    (`sp.is_paid && !sp.inactive`) auf Modulebene neben `compute_status` hinzufügen.
  - Beide `for sp in props.sales_persons.iter()`-Loops auf
    `.filter(|sp| is_selectable_employee(sp))` umstellen.
- **verify:** `cargo build --target wasm32-unknown-unknown` (WASM-Gate) + `cargo test`
- **done:** Beide Dropdowns zeigen nur bezahlte & aktive Personen; Helper-Funktion existiert.

### Task 2: Unit-Tests für den Helper

- **action:** Tests für alle vier Kombinationen (paid×active) in der bestehenden

- **verify:** `cargo test` grün.
- **done:** Filterlogik durch Tests abgedeckt.
